# pmo-today — 今日概览

## 执行方式

```bash
# 当前项目今日概览
claude pmo-today

# 跨所有关注项目（或全部 active 项目）
claude pmo-today --all
```

## 前置条件

已通过 `pmo-use` 设置当前项目（`--all` 模式不需要）。

## 执行流程

### 公共：待处理队列检查

执行前先检查以下目录（按 CLAUDE.md 公共约定）：

| 目录 | 用途 | 处理方式 |
|------|------|---------|
| `~/.smart-pmo/.pending_backfill/` | 会议索引回填失败 | 自动重试回填，成功删文件 |
| `~/.smart-pmo/.pending_orphan_meeting/` | 孤立会议记录（步骤②成功+步骤③全部失败）| 提示用户执行 `--index-only` 补录 |
| `~/.smart-pmo/.pending_assignee/` | 负责人 API 写入失败 | 提示用户存在待分配记录 |
| `~/.smart-pmo/.draft/` | 用户取消的解析草稿 | 提示用户存在缓存草稿 |

过期清理规则见 CLAUDE.md「待处理队列过期清理规则」。

### 配置加载

1. 按 CLAUDE.md「读取当前项目配置」规则加载项目配置（--all 模式：遍历 pinned 或所有 active 项目）
2. 检查 `schemaVersion`，执行必要的版本迁移
3. 执行配置完整性校验（必填字段：`project.name`、`larkResources.baseAppToken`、`larkResources.baseTableIds.*`）

### 第1步：确定目标项目列表

- 默认：读取 `~/.smart-pmo/current` 加载当前项目
- `--all`：读取 `~/.smart-pmo/pinned`，若为空则使用所有 `status=active` 的项目

### 第2步：并行查询各项目 Base

对每个目标项目，从 Base 并行查询以下数据（`currentDate` = 当日日期）：

| 数据项 | 查询条件 |
|--------|---------|
| 今日截止待办 | 截止日期 = currentDate 且 状态 ∈ {待处理, 进行中} |
| 已过期待办 | 截止日期 < currentDate 且 状态 ∈ {待处理, 进行中} |
| 近7天到期里程碑 | 计划日期 ≤ currentDate+7 且 状态 ∉ {已完成, 已取消} |
| 今日会议 | 会议日期 = currentDate |

### 第3步：展示今日概览

**单项目格式：**

```
📅 今日概览 — {项目名} · {currentDate}
══════════════════════════════════════

⏰ 今日截止（{N} 项）:
  1. [TODO-003] 确定UI方案 | @李四 | P0-紧急
  2. [TODO-007] 发送测试报告 | @张三 | P2-一般

⚠️ 已过期（{N} 项）:
  3. [TODO-001] 接口文档更新 | @王五 | 截止 06-08 ‼️
  4. [TODO-005] 用例评审 | @赵六 | 截止 06-05 ‼️

🏁 本周到期里程碑（{N} 项）:
  · [MILE-002] Beta版发布 | @张三 | 06-13（还有2天）
  · [MILE-004] UAT评审 | @李四 | 06-15（还有4天） ⚠️ 进度 30%

📝 今日会议（{N} 场）:
  · Sprint评审 · 14:00（已有纪要: [查看]）
  · 架构讨论 · 16:00（待处理）

─────────────────────────────────
快速操作：
  pmo-todo-followup --complete <序号>    标记完成
  pmo-todo-followup --modify <序号> --due <日期>  推迟截止日期
  pmo-meeting-process --text             处理今日会议记录
```

**`--all` 模式格式（按项目分组）：**

```
📅 今日概览 · {currentDate} — {N} 个项目
══════════════════════════════════════════

◆ {项目名}（{代号}）
  今日截止 {N} | 过期 {N} | 里程碑即将到期 {N} | 今日会议 {N}
  ─ 今日截止: [TODO-003] 确定UI方案 @李四 · [TODO-007] 发送测试报告 @张三
  ─ 过期: [TODO-001] 接口文档更新 @王五（06-08）

◆ {项目名2}（{代号2}）
  今日截止 0 | 过期 2 | 里程碑即将到期 1 | 今日会议 0
  ─ 过期: [TODO-012] ... @赵六（06-07）

─────────────────────────────────
汇总：今日截止 {总N} | 过期 {总N} | 里程碑到期 {总N}
使用 pmo-use <项目名> 后操作指定项目，或 pmo-todo-followup --all --overdue 查看全部过期
```

**无任何需要关注项时：**

```
📅 今日概览 — {项目名} · {currentDate}
══════════════════════════════════════
✅ 今天没有需要特别关注的事项

  待处理待办: {N} 项（使用 pmo-todo-followup 查看）
  进行中里程碑: {N} 项
```

## 异常处理

| 场景 | 处理 |
|------|------|
| 无当前项目（非 --all 模式）| 提示运行 pmo-use |
| 单个项目 Base 查询失败 | 该项目显示 ⚠️ 查询失败，不影响其他项目 |
| --all 且无任何项目 | 提示"无已注册项目，请先运行 pmo-init" |
| 所有项目 Base 查询失败 | 展示失败汇总 + 排查建议（登录状态、Base 权限、网络） |

## 边缘情况

| 场景 | 处理方式 |
|------|---------|
| 今日无任何关注项 | 显示待处理总数 + 进行中里程碑数 |
| 今日有会议但无纪要 | 标注「待处理」，提示 pmo-meeting-process |
| 今日有会议已有纪要 | 标注「已有纪要: [查看]」 |
| 里程碑即将到期且进度低（<30%）| 额外标注 ⚠️ 进度 |
| --all 时 pinned 为空 | 回退到所有 active 项目 |
| 单项目 Base 查询失败 | 标注 ⚠️，其余正常 |
