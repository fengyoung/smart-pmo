# pmo-today — 今日概览

## 执行方式

```bash
# 查看当前项目今日概览
claude pmo-today

# 查看所有关注项目今日概览
claude pmo-today --all
```

## 前置条件

已通过 `pmo-use` 设置当前项目（`--all` 模式不需要）。

## 执行流程

### 第1步：获取项目列表

- 默认：当前项目
- `--all`：读取 `~/.smart-pmo/pinned`，若为空则使用所有 active 项目

### 第2步：并行查询各项目

对每个目标项目，通过 `lark-base` 并行查询：

1. **今日截止待办**：截止日期 = currentDate，状态 ≠ 已完成/已取消
2. **过期待办**：截止日期 < currentDate，状态 ≠ 已完成/已取消
3. **即将到期里程碑**：计划日期在 [currentDate, currentDate+7] 之间，状态 ≠ 已完成/已取消
4. **已过期里程碑**：计划日期 < currentDate，状态 ≠ 已完成/已取消
5. **今日会议**：会议日期 = currentDate

### 无关注项时

```
📅 今日概览 · {currentDate} ({星期X})
═══════════════════════════════════

✅ 今日无过期待办、无即将到期里程碑、无会议安排！

使用 pmo-dashboard 查看完整项目状态
```

### `--all` 模式

每个项目独立展示一个区块，按告警严重程度排序（过期待办数 > 今日截止数 > 里程碑到期数）。

## 异常处理

| 场景 | 处理 |
|------|------|
| 无当前项目且未加 --all | 提示运行 pmo-use 或加 --all |
| Base 连接失败 | 对应项目区块显示 ⚠️ 连接失败 |
| 所有项目无关注项 | 显示成功消息 |
