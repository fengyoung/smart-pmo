# pmo-stats — 统计分析

> 基于 Base 记录的时间戳数据，按周/月维度统计待办趋势和团队效率。

## 执行方式

```bash
# 当前项目近 4 周统计
claude pmo-stats

# 指定统计周期数
claude pmo-stats --weeks 8

# 按月统计
claude pmo-stats --monthly --months 3

# 指定项目
claude pmo-stats --project XRay

# 跨所有关注项目汇总
claude pmo-stats --all

# 导出统计报告为 Markdown
claude pmo-stats --export stats-report.md
```

## 前置条件

- 当前项目或 `--project` 指定的项目 Base 中有数据
- 项目创建至少有一段时间的数据积累

## 执行流程

### 第1步：确定统计维度

```
周期类型 = --monthly ? 月 : 周
周期数   = --weeks（默认 4）/ --months（默认 3）
目标项目 = --project / --all / 当前项目
```

### 第2步：拉取 Base 数据

对每个目标项目的待办表和里程碑表，拉取所有记录（增量拉取，需要创建时间和完成时间字段）：

- 待办：读取所有记录的创建时间、完成时间、状态、负责人
- 里程碑：读取所有记录的计划日期、实际日期、状态

### 第3步：按周期聚合统计

## 异常处理

| 场景 | 处理 |
|------|------|
| 项目数据不足 2 周 | 提示「数据积累不足，至少需要 2 周的数据才能生成趋势」 |
| Base 无完成时间记录 | 完成数基于当前状态估算，标注 `*估算` |
| 负责人字段为空 | 统一归类为「未分配」 |
| --all 中某项目失败 | 跳过该项目，汇总中标注 |

## 边缘情况

| 场景 | 处理方式 |
|------|---------|
| 新项目（创建 < 7 天）| 提示「项目刚启动，建议运行满 2 周后再查看统计」 |
| 指定周期内无数据 | 对应行显示 `-`，不纳入趋势计算 |
| 积压为 0 | 展示 `🎉 暂无积压` |
