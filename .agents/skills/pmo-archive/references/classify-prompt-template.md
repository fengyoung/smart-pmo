# pmo-archive — 文档分类 Prompt 模板

> LLM 调用模板，用于对文档内容进行语义理解并分类到知识库目录。
> 采用标准三段式 Prompt 设计：背景上下文 → 任务说明 → 输出格式。

---

## 完整 Prompt 结构

```
SYSTEM_PROMPT: 你是一个专业的文档分类专家，擅长根据文档内容语义判断其所属的类别。
当前日期：{currentDate}
注意：只输出分类结果，不要输出任何前缀说明、开场白或分析过程。严格按指定格式输出。

--- 背景上下文 ---

## 项目信息
项目名称：{projectName}（{projectAlias}）
知识库有以下标准目录：

{wikiDirectories}

## 文档信息
文件名：{filename}
来源：{sourceType}（{sourceDetail}）
{extraMeta}

## 文档内容（用于分类的前 {maxContentChars} 字符）
{contentSnippet}

--- 任务说明 ---

根据文档内容，判断该文档应该归档到以下哪个目录：

{wikiDirectoryDescriptions}

分类判断依据（按优先级）：
1. 文档标题和正文中的关键主题词
2. 文档的结构特征（如包含"待办事项""会议日期"→会议纪要；包含"需求概述""用户故事"→需求文档；包含"架构图""技术方案"→设计文档）
3. 文档的用途和受众
4. 文件名中的关键词（仅作辅助参考）

置信度判断：
- high：文档主题非常明确，与某目录高度匹配（如典型会议纪要 → 01-会议纪要）
- medium：文档主题较明确，但有一定歧义（如在"需求文档"和"设计文档"之间）
- low：文档内容不够清晰，或可能匹配多个目录

RENAME 建议（可选）：
- 若文件名不能反映文档实际内容 → 建议一个更合适的文件名（不含日期前缀）
- 若文件名已足够描述 → 留空

--- 输出格式 ---

严格按以下格式输出，每行一个字段，不要输出其他内容：

DIR: <目录名（如 01-会议纪要）>
CONFIDENCE: high | medium | low
REASON: <一句话归类理由，用中文，不超过50字>
RENAME: <建议的新文件名（不含日期前缀），如不需要则留空>
```

---

## 变量说明

| 变量 | 来源 | 说明 |
|------|------|------|
| `{currentDate}` | 系统 | 当前日期，供上下文参考 |
| `{projectName}` | 项目配置 | `project.name` |
| `{projectAlias}` | 项目配置 | `project.alias` |
| `{wikiDirectories}` | 项目配置 | `wikiNodeTokens` 的 keys 排序后列表，如 `01-会议纪要, 02-周报, 03-需求文档, 04-设计文档, 05-项目资料, 99-归档` |
| `{wikiDirectoryDescriptions}` | 固定映射 | 每个目录的用途说明（见下方） |
| `{filename}` | 输入文件 | 文件名（含扩展名） |
| `{sourceType}` | 输入源 | `本地文件` 或 `飞书文档` |
| `{sourceDetail}` | 输入源 | 本地路径 或 飞书链接 URL |
| `{extraMeta}` | 输入源 | 飞书文档时有：`文档标题: {title}\n飞书链接: {url}`；本地文件时为空 |
| `{contentSnippet}` | Step 1 | 提取的文本内容，截断到 `{maxContentChars}` 字符 |
| `{maxContentChars}` | 固定 | 3000（确保不超过 token 限制，同时保留足够分类信息） |

---

## 目录用途描述（`{wikiDirectoryDescriptions}`）

```
01-会议纪要：会议记录、讨论纪要、语音转写、评审记录、站会记录
02-周报：周报、双周报、月报、季度总结、项目进度报告
03-需求文档：PRD、需求说明、用户故事、功能规格、产品方案、业务分析
04-设计文档：技术方案、架构设计、接口文档、数据库设计、流程图、UML
05-项目资料：项目计划、排期表、人员名单、参考材料、规范标准、ASR校正表、工具说明
99-归档：历史文档、已废弃内容、通用资料、无法归类的文档
```

---

## LLM 参数建议

| 参数 | 值 | 说明 |
|------|-----|------|
| temperature | 0.0 | 分类任务需要最高一致性，不需要创造性 |
| max_tokens | 256 | 输出极短（仅 4 行），无需大 token 预算 |

## 使用示例

**输入：**
```
文件名：20260618-项目讨论-数据标注平台优化与业务流程迭代.txt
来源：本地文件（/Users/.../20260618-项目讨论-...txt）
内容：（前3000字符）...会议讨论...标注平台...五月版本上线...待办事项...
```

**LLM 输出：**
```
DIR: 01-会议纪要
CONFIDENCE: high
REASON: 典型的项目讨论会议记录，含议题讨论和待办事项
RENAME:
```

---

**输入：**
```
文件名：数据接入流程技术方案v2.docx
来源：飞书文档（https://xxx.feishu.cn/docx/xxx）
文档标题：数据接入流程技术方案 v2.0
内容：（前3000字符）...架构设计...Hive取数...自动处理...接口定义...
```

**LLM 输出：**
```
DIR: 04-设计文档
CONFIDENCE: high
REASON: 技术架构方案文档，涉及系统接口和数据处理流程设计
RENAME:
```

---

> **注意：** 图像文件（.png/.jpg）无可提取文本，跳过 AI 分类，走文件名关键词规则（见 SKILL.md「图像文件处理」章节）。此模板不用于图像文件。
