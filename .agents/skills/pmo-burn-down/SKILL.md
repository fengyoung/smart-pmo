# pmo-burn-down — 燃尽图

> 选择目标里程碑，基于关联待办的完成进度，在终端渲染 ASCII 燃尽图，对比理想线与实际线，量化项目偏差风险。

## 执行方式

```bash
# 列出进行中里程碑供选择
claude pmo-burn-down

# 指定里程碑 ID
claude pmo-burn-down --milestone MILE-001

# 展示所有进行中里程碑的概览（一行一个）
claude pmo-burn-down --all-milestones

# 指定项目（无需切换上下文）
claude pmo-burn-down --project XRay
```

## 前置条件

已通过 `pmo-use` 设置当前项目（或使用 `--project` 指定）。

目标里程碑需在 Base 中通过「关联待办」字段关联了待办记录；若未关联，燃尽图仍可展示，但显示「里程碑无关联待办」提示。

## 执行流程

### 公共：待处理队列检查

> 📋 详见 [`_shared/pending-queue-check.md`](../_shared/pending-queue-check.md)。

### 配置加载

1. 按 CLAUDE.md「读取当前项目配置」规则加载项目配置
2. 检查 `schemaVersion`，执行必要的版本迁移
3. 执行配置完整性校验（必填字段：`project.name`、`larkResources.baseAppToken`、`larkResources.baseTableIds.todos`、`larkResources.baseTableIds.milestones`）

### 第1步：选择目标里程碑

**指定了 `--milestone <ID>`：** 直接使用，若 ID 不存在则提示错误。

**未指定 ID（默认）：** 通过 `lark-base` 查询所有状态≠已完成/已取消的里程碑，展示交互列表：

```
请选择目标里程碑：
  1. [MILE-001] Beta版发布    | 计划 06-01 ~ 06-15 | 进行中 | 关联待办 8 项
  2. [MILE-002] UAT评审       | 计划 06-08 ~ 06-20 | 进行中 | 关联待办 3 项
  3. [MILE-003] 正式上线      | 计划 06-21 ~ 06-30 | 未开始 | 无关联待办
请输入序号：
```

**`--all-milestones` 模式：** 对所有进行中里程碑并行执行燃尽计算，每个里程碑输出一行摘要（见第4步格式）。

### 第2步：拉取里程碑与关联待办

通过 `lark-base` 查询：

1. **里程碑记录**（含计划日期、实际完成日期、状态、关联待办 ID 列表）
2. **关联待办记录**（逐条拉取，含创建时间、完成日期、状态）

**数据点计算**：
- `startDate`：里程碑计划开始日（若无单独的开始日期字段，使用创建时间）
- `planDate`：里程碑计划完成日期
- `totalTodos`：关联待办总数（燃尽图 Y 轴最大值）

**若里程碑无关联待办：**
```
⚠️ [MILE-001] Beta版发布 无关联待办，无法生成燃尽图。
→ 通过 pmo-milestone --modify MILE-001 关联待办后重试。
```

### 第3步：计算理想线与实际线

**理想线（Ideal Burndown）：**
- 从 `(startDate, totalTodos)` 到 `(planDate, 0)` 的线性插值
- 每天应完成的待办数 = `totalTodos / 总天数`
- 理想剩余(day) = `totalTodos - totalTodos * (day - startDate) / (planDate - startDate)`

**实际线（Actual Burndown）：**
- 按每日累计统计已完成待办数
- 若某日无完成 → 继承前一日累计值
- 仅统计到 `min(today, planDate)` 的数据

**偏差计算：**
- 今日理想剩余 vs 今日实际剩余
- 偏差率 = `(实际剩余 - 理想剩余) / totalTodos × 100%`
- 偏差 > 20% → 🔴 偏离较大；> 10% → 🟡 轻微落后；≤ 10% → 🟢 按计划

**预计完成日期推算：**
- 基于最近 7 天的平均完成速率（条/天）
- 若速率 > 0：`预计完成日 = today + ceil(实际剩余 / 近7日平均速率)`
- 若速率 = 0（无进展）：显示「⚠️ 近期无进展，无法推算」

### 第4步：渲染 ASCII 燃尽图

**单里程碑完整图（默认模式）：**

```
🔥 燃尽图 — {里程碑名称}（{MILE-ID}）
计划：{startDate} → {planDate}   总任务：{N} 项
今日：{today}   已完成：{done}/{N}（{pct}%）

剩余
{N} │◆ · · · · · · · · · · · ·
    │ ╲ · · · · · · · · · · ·     — 理想线
{m} │  ·╲· · · · · · · · ·
    │  · ·╲· · · · ◆─────       ← 实际线（{today}）
    │  · · · ╲· · · ·
 0  └──────────────────── 日期
    {s}       {m}       {e}

{偏差状态}：{🔴 偏离理想线 {pct}%，建议加速推进 | 🟡 轻微落后 | 🟢 按计划推进}
预计完成：{估算日期}（{超出/剩余} {N} 天）

关联待办状态：
  已完成 {done} 项 | 进行中 {wip} 项 | 待处理 {todo} 项
```

**图形渲染规则：**
- Y 轴：0 ~ totalTodos，按比例缩放到 10 行高
- X 轴：startDate ~ planDate，按比例缩放到 40 列宽
- 理想线用 `·` 表示，实际线用 `◆` 和 `─` 表示
- 今日位置用 `←` 标注
- 超过 planDate 的实际线段用 `🔴` 标注（延期区间）

**`--all-milestones` 摘要模式（一行一个）：**

```
🔥 燃尽图概览 · {today}
══════════════════════════════

里程碑            总计  完成  剩余  偏差   状态    预计完成
──────────────────────────────────────────────────────────
MILE-001 Beta发布  8    6     2    -12%  🟢按计划  06-14
MILE-002 UAT评审   3    1     2    +28%  🔴偏离    06-22 (+2天)
```

## 异常处理

| 场景 | 处理方式 |
|------|------|
| 无当前项目且未指定 --project | 提示「请先执行 pmo-use <项目名>」 |
| 里程碑 ID 不存在 | 提示「找不到里程碑 {ID}，请通过 pmo-milestone 查看」 |
| 里程碑无计划日期 | 提示「里程碑 {ID} 缺少计划日期，无法绘制燃尽图」 |
| 里程碑无关联待办 | 展示提示 + 建议操作（见第2步） |
| startDate > today | 提示「里程碑尚未开始，暂无实际数据」，展示理想线预览 |
| 全部待办已完成 | 显示「✅ 所有关联待办已完成！」+ 最终燃尽图 |
| Base 查询失败 | 提示 ⚠️ 查询失败 + 具体错误，建议检查网络/权限 |
| --all-milestones 无进行中里程碑 | 提示「当前无进行中里程碑」 |

## 边缘情况

| 场景 | 处理方式 |
|------|------|
| 完成速率为 0（近 7 天无完成）| 预计完成日显示「⚠️ 近期无进展，无法推算」 |
| 里程碑已延期（today > planDate）| X 轴延伸至 today，延期区间用 `🔴` 标注 |
| 总待办 < 5 项 | 燃尽图 Y 轴调整为 5 行高，避免过于稀疏 |
| 单天完成数异常高（>总数 20%）| 标注 ⚡ 突击完成日 |
