# pmo-meeting-prep — 会前准备

> 自动汇总上次会议未完成事项和当前项目风险点，生成会前议程文档，让会议主持人 5 分钟完成准备。

## 执行方式

```bash
# 为当前项目生成会前议程
claude pmo-meeting-prep

# 指定会议日期（默认自动取最近一次会议的下一个周期）
claude pmo-meeting-prep --date 2026-06-16

# 仅预览议程，不归档
claude pmo-meeting-prep --dry-run

# 生成议程并推送到项目群
claude pmo-meeting-prep --send

# 自定义会议主题
claude pmo-meeting-prep --topic "Sprint 评审 + 风险盘点"
```

## 执行流程

### 第1步：确定上次会议和议程范围

1. 查询 Base「会议记录索引」表，按会议日期倒序，取最近一次会议
2. 若无历史会议记录 → 提示「首次会议，将基于当前待办和里程碑生成议程」
3. 读取上次会议关联的「产出待办」，筛选未完成的项
4. 若未指定 `--date`，议程日期默认为上次会议 + 7 天（同周期）

### 第2步：收集议程素材

并行查询：

| 素材 | 查询方式 | 用途 |
|------|---------|------|
| 上次未完成待办 | 通过会议索引的「产出待办」关联查询，状态≠已完成/已取消 | 跟进项 |
| 上次关键决策 | 读取会议索引的「关键决策」字段 | 决策回顾 |
| 过期待办 | 截止日期 < currentDate，状态≠已完成/已取消 | 风险项 |
| 临近到期里程碑 | 计划日期在 14 天内，状态≠已完成/已取消 | 进度检查 |
| 无进展待办 | 状态=待处理 且 创建时间 > 14 天 | 推动项 |

### 第3步：生成议程文档

按以下结构生成飞书文档：

```
📋 {项目名} 会议议程 · {日期}

── 一、上次会议回顾 ──
会议：{上次会议主题}（{日期}）
关键决策：
  · {决策1}
  · {决策2}

未完成待办：
  □ {待办1} — @{负责人}（截止 {日期}，已逾期 {N} 天）
  □ {待办2} — @{负责人}
  ✅ {已完成待办}（已完成）

── 二、本次议题 ──
1. 上次遗留跟进（{N} 项未完成）
2. 里程碑进度检查（{N} 项临近到期）
3. 风险事项（{N} 条过期待办）
4. 长期无进展事项推动（{N} 条超 14 天）

── 三、待讨论决策 ──
（预留，会议中补充）

── 四、里程碑概览 ──
🏁 进行中：{N} 项
⚠️ 14天内到期：{N} 项
  · {里程碑名称} — 计划 {日期}，进度 {X}%
```

### 第4步：归档与推送

1. 通过 `lark-doc` 创建飞书文档
2. 归档到 `config.larkResources.wikiNodeTokens["01-会议纪要"]` 对应的目录节点（文件名：`{YYYYMMDD}-{会议主题}-议程`）
3. 若指定 `--send`：推送议程卡片到项目群
4. **追溯性**：将议程文档链接写入 Base「会议记录索引」表中（若已有对应会议记录则更新「纪要文档链接」字段；若无则创建一条新记录，会议主题后加"（议程）"标注，状态与正式会议纪要区分）

**推送卡片：**
```
📋 {项目名} 会议预告 · {日期}
  主题：{会议主题}
  
  上次遗留：{N} 项 | 过期待办：{N} 条
  里程碑到期：{N} 项 | 无进展项：{N} 条

  📎 完整议程 → {wiki链接}
```

### 第5步：输出确认

```
✅ 会前议程已生成
═══════════════════════════════
  文档：20260616-项目周会-议程
  归档：01-会议纪要/
  链接：{wiki文档链接}
  
  议程亮点：
  · 上次 {N} 项未完成待办全部纳入跟进
  · {N} 条过期待办标注为风险项
  · {N} 项里程碑 14 天内到期
```

## 异常处理

| 场景 | 处理 |
|------|------|
| 无历史会议记录 | 生成仅含当前待办/里程碑的议程，标注「首次会议」 |
| 上次会议无未完成待办 | 跟进项显示 `✅ 全部完成` |
| 所有待办均正常 | 风险项显示 `无` |
| Base 查询失败 | 仅生成框架文档，内容标注 `*数据获取失败` |

## 边缘情况

| 场景 | 处理方式 |
|------|---------|
| 上次会议距今 > 30 天 | 标注 ⚠️ 会议间隔过长，建议缩小回顾范围 |
| 项目无里程碑 | 跳过里程碑章节 |
| 生成后用户取消推送 | 文档仍归档到知识库 |
