# pmo-pin / pmo-unpin — 项目关注管理

## 前置条件

无。pmo-pin 仅操作本地文件 `~/.smart-pmo/pinned`，不依赖飞书 API。

## 执行方式

```bash
# 关注一个或多个项目
claude pmo-pin <项目名> [<项目名>...]

# 取消关注
claude pmo-unpin <项目名> [<项目名>...]
```

## 执行流程

> 本 Skill 仅执行纯本地文件操作，无需检查待处理队列或调用飞书 API。

### pmo-pin

1. 解析命令行参数，获取要关注的项目名列表（支持空格分隔多个项目名）
2. 对每个项目名，校验项目存在性：
   - 读取 `~/.smart-pmo/registry/` 目录下所有 `.json` 文件
   - 按文件名（不含 `.json`）精确匹配
   - 若未匹配 → 再遍历所有 registry JSON，按 `project.name` 全称匹配
   - 若仍未匹配 → 提示 `⚠️ 项目 "{项目名}" 未注册，请先执行 pmo-init 创建项目`，跳过该项
3. 读取 `~/.smart-pmo/pinned` 文件：
   - 文件不存在 → 视为空列表
   - 解析为行数组，过滤空行
4. 将要关注的有效项目名加入列表（去重：已存在的不重复添加）
5. 写回 `~/.smart-pmo/pinned` 文件（每行一个项目名）
6. 输出确认

**输出格式：**

```
📌 已关注项目
═══════════════════════════════
  ✅ XRay（XRay拆修检测2026） — 已在关注列表中
  ✅ RCA（新增关注）
───────────────────────────────
当前关注 {N} 个项目：XRay, RCA

使用 pmo-dashboard 查看概览
```

**新增/已存在区分展示，已存在的项目标注「已在关注列表中」。**

### pmo-unpin

> pmo-unpin 有独立的 SKILL.md 注册。若以 `pmo-unpin` 调用，执行以下流程：

1. 解析命令行参数，获取要取消关注的项目名列表
2. 读取 `~/.smart-pmo/pinned` 文件
3. 从列表中移除指定项目名（不存在的项目名静默忽略）
4. 写回文件
5. 输出确认

**输出格式（与 pmo-pin 风格一致）：**

```
📌 已取消关注
═══════════════════════════════
  ❌ XRay — 已移除
───────────────────────────────
当前关注 {N} 个项目：RCA

使用 pmo-pin <项目名> 可重新关注
```

## 存储格式

`~/.smart-pmo/pinned` 为纯文本，每行一个项目名（project_id = registry 文件名不含 `.json`）：

```
XRay
RCA
```

## 边缘情况

| 场景 | 处理方式 |
|------|---------|
| 未传项目名 | 显示当前关注列表 + 使用提示 |
| pinned 文件不存在 | 视为空列表，正常创建 |
| 项目名未注册 | ⚠️ 提示未注册，跳过该项，其余正常处理 |
| 重复关注 | 静默跳过，输出中标注「已在关注列表中」 |
| 取消关注未关注的项目 | 静默跳过 |
| pinned 全部移除后为空 | 正常写入空文件，提示「当前无关注项目，pmo-dashboard 将展示所有 active 项目」 |
