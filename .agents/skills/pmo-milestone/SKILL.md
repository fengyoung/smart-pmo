# pmo-milestone — 里程碑管理

## 执行方式

```bash
# 查看所有里程碑（附行序号）
claude pmo-milestone

# 检查到期情况（7天内到期 + 已过期）
claude pmo-milestone --check

# 新增里程碑
claude pmo-milestone --add "里程碑名称" --due YYYY-MM-DD --owner @姓名

# 修改里程碑（支持 MILE-ID 或查看列表后的行序号）
claude pmo-milestone --modify MILE-001 --due 2026-07-15
claude pmo-milestone --modify 2 --due 2026-07-15
claude pmo-milestone --modify MILE-001 --owner @王五
claude pmo-milestone --modify MILE-001 --progress 75
claude pmo-milestone --modify MILE-001 --status 已延期

# 标记完成（支持 MILE-ID 或行序号）
claude pmo-milestone --complete MILE-001
claude pmo-milestone --complete 3
```

## 前置条件

已通过 `pmo-use` 设置当前项目。

**所有 Base 写操作遵循公共错误重试策略（见 CLAUDE.md）：3 次指数退避重试（1s/3s/5s）。**

## 执行流程

### 公共：待处理队列检查

> 📋 详见 [`_shared/pending-queue-check.md`](../_shared/pending-queue-check.md)。执行开始时检查 `~/.smart-pmo/` 下的四个待处理目录。

### 配置加载

1. 按 CLAUDE.md「读取当前项目配置」规则加载项目配置
2. 检查 `schemaVersion`，执行必要的版本迁移
3. 执行配置完整性校验（必填字段：`project.name`、`larkResources.baseAppToken`、`larkResources.baseTableIds.milestones`）

### 查看所有

从 Base「里程碑」表读取所有记录，按状态分组展示，**每行附带行序号**（从 1 开始，跨分组连续）：

```

═══════════════════════════════════

┌──┬──────────┬──────────────────┬──────────┬──────────┬──────┐
│# │ ID       │ 名称              │ 负责人   │ 计划日期  │ 进度 │
├──┼──────────┼──────────────────┼──────────┼──────────┼──────┤
│1 │ MILE-001 │ Beta版发布        │ 张三     │ 06-15    │  30% │
│2 │ MILE-003 │ API接口冻结       │ 李四     │ 06-20    │  60% │
└──┴──────────┴──────────────────┴──────────┴──────────┴──────┘

未开始（{N} 项）:
┌──┬──────────┬──────────────────┬──────────┬──────────┬──────┐
│3 │ MILE-004 │ 用户测试          │ 王五     │ 07-01    │   0% │
└──┴──────────┴──────────────────┴──────────┴──────────┴──────┘

┌──┬──────────┬──────────────────┬──────────┬──────────┬──────────┬──────┐
│4 │ MILE-002 │ 需求评审          │ 张三     │ 05-20    │ 05-20    │ 100% │
└──┴──────────┴──────────────────┴──────────┴──────────┴──────────┴──────┘

操作提示（可用序号或 MILE-ID）：
  pmo-milestone --complete 1            标记第1条完成
  pmo-milestone --modify 2 --progress 80
```

**已完成分组默认折叠**：已完成/已取消的里程碑仅显示最近 5 条，超过时提示「... 还有 {N} 条已完成记录，使用 pmo-milestone --all 查看全部」。

**空列表展示：**

```
🏁 里程碑列表 — {项目名} (0 项)
═══════════════════════════════════
暂无里程碑记录

使用 pmo-milestone --add "名称" --due YYYY-MM-DD --owner @姓名 新增
```

### 参数解析（--modify / --complete）

- 纯数字（如 `2`、`3`）→ 解析为列表中的行序号，映射到对应 MILE-ID
- `MILE-XXX` 格式 → 直接使用该 ID

**行序号生命周期（⚠️ 重要）：**
行序号仅在**当次展示的列表**中有效，切换项目或任何写操作后序号均失效。如果使用行序号但本次会话中未先查看列表，**自动先重新执行一次查看**，再继续操作，并提示：
```
ℹ️ 序号映射已刷新（Base 可能已变更），以最新列表为准
```

### 到期检查 --check

1. 读取所有状态≠已完成/已取消的里程碑
2. 筛选即将到期（计划日期在 currentDate ~ currentDate+7 天之间）和已过期（计划日期 < currentDate）
3. 在终端输出检查结果

**输出格式：**

```
⚠️ 里程碑到期检查 — {projectName} · {currentDate}
══════════════════════════════════════════════
🔴 已过期（{N} 项）:
  · MILE-001 Beta版发布 — 计划 06-01，已逾期 {N} 天 — @张三
  · MILE-003 API接口冻结 — 计划 06-08，已逾期 {N} 天 — @李四

🟡 即将到期（{N} 项，7天内）:
  · MILE-005 用户测试 — 计划 06-15，剩余 {N} 天 — @王五

──────────────────────────────────────────────
共 {N} 项需要关注

操作提示：
  pmo-milestone --modify <ID> --status 已延期   标记延期
  pmo-milestone --modify <ID> --due <新日期>    调整计划日期
```

**无告警时：**

```
✅ 里程碑到期检查 — {projectName} · {currentDate}
══════════════════════════════════════════════
所有里程碑状态正常，无过期或即将到期项
```

### 新增 --add

1. 解析 `--owner @姓名`：
   - 在项目配置 `team.members` 中按姓名精确匹配 openId
   - 找不到 → 通过 `lark-contact` 搜索飞书通讯录
   - 仍找不到 → 负责人字段留空，提示 `⚠️ @{姓名} 未匹配，请手动指定`
2. 解析 `--due YYYY-MM-DD`：截止日期（必填）
3. 可选参数：
   - `--progress 0-100`：初始进度，默认 0
   - `--status <状态值>`：初始状态，默认「未开始」
   - `--desc "描述文本"`：里程碑描述
4. 通过 `lark-base` 写入里程碑表（负责人字段按 `[{"id": "ou_xxx", "name": "姓名"}]` 格式写入）
5. 错误重试：遵循公共错误重试策略
6. 输出确认：

```
✅ 里程碑已创建
──────────────────────────────
  ID：MILE-005
  名称：用户测试
  计划日期：2026-07-01
  负责人：@王五
  状态：未开始
```

### 修改 --modify

1. 按 ID（或行序号映射后的 ID）查找里程碑记录
2. 更新指定字段：

   | 参数 | 说明 | 校验规则 |
   |------|------|---------|
   | `--due YYYY-MM-DD` | 修改计划日期 | 格式校验 YYYY-MM-DD |
   | `--owner @姓名` | 修改负责人 | 走成员名称解析流程 |
   | `--progress 0-100` | 修改进度 | 0~100 整数 |
   | `--status <状态值>` | 修改状态 | 必须为有效值（见下方） |
   | `--desc "描述文本"` | 修改描述 | — |

3. **状态值校验**：`--status` 仅接受以下值：
   - `未开始` / `进行中` / `已完成` / `已延期` / `已取消`
   - 无效值 → 提示 `⚠️ 无效状态 "{值}"，可选: 未开始 / 进行中 / 已完成 / 已延期 / 已取消`，中断操作

4. 通过 `lark-base update_record` 更新，错误重试遵循公共策略
5. 输出确认：

```
✅ MILE-001 已更新
──────────────────────────────
  进度：30% → 75%
  状态：进行中 → 进行中
```

### 标记完成 --complete

1. 按 ID（或行序号映射后的 ID）查找里程碑记录
2. `lark-base` 更新：
   - 状态 = "已完成"
   - 实际日期 = currentDate
   - 进度 = 100（若当前进度 < 100，自动补齐）
3. 错误重试遵循公共策略
4. 输出确认：

```
✅ MILE-001 Beta版发布 已标记完成
──────────────────────────────
  实际完成日期：2026-06-12
```

## 边缘情况

| 场景 | 处理方式 |
|------|---------|
| 无当前项目 | 提示「请先执行 pmo-use <项目名>」 |
| 无里程碑记录 | 显示空列表 + 新增提示 |
| 新增时负责人未匹配 | 留空并标注 ⚠️，不阻塞写入 |
| 用序号但未先查看 | 自动先查看列表再操作 |
| 无效序号（超出范围） | 提示「⚠️ 序号 {N} 无效，当前列表共 {M} 条」 |
| MILE-ID 不存在 | 提示「⚠️ {MILE-ID} 不存在」，中断操作 |
| 无效状态值 | 提示可选值列表，中断操作 |
| 无效进度值 | 提示「⚠️ 进度需为 0~100 的整数」 |
| Base 写入失败（重试耗尽） | 提示「❌ 写入失败，请稍后重试或手动处理」，记录错误详情 |
