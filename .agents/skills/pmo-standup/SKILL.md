# pmo-standup — 每日站会速报

> 站会前 30 秒生成当日速报，聚焦个人视角：昨天做了什么、今天计划什么、有什么阻塞。输出格式适合直接分享到群聊。

与 `pmo-today` 的区别：
- `pmo-today`：全员视角，展示所有人的待办/里程碑，用于浏览
- `pmo-standup`：**个人视角**，基于当前用户负责的待办，输出可分享文本

## 执行方式

```bash
# 生成个人站会速报（基于当前登录用户）
claude pmo-standup

# 生成并直接推送到项目群
claude pmo-standup --send

# 跨所有关注项目的个人站会汇总
claude pmo-standup --all

# 查看指定成员的站会速报（PM 代替团队成员生成）
claude pmo-standup --member 张三

# 查看所有成员的站会速报（适合 PM 在站会上主持时使用）
claude pmo-standup --team
```

## 前置条件

已通过 `pmo-use` 设置当前项目。

**Base 读操作超时遵循 CLAUDE.md 公共配置（单次 20s，并发 30s）。所有飞书 API 写操作（IM 推送）遵循公共错误重试策略：3 次指数退避重试（1s/3s/5s）。**

## 执行流程

### 公共：待处理队列检查

执行前检查待处理队列（仅在有积压时简短提示，不阻塞主流程），详见 [`_shared/pending-queue-check.md`](../_shared/pending-queue-check.md)。

### 配置加载

1. 按 CLAUDE.md「读取当前项目配置」规则加载项目配置
2. 检查 `schemaVersion`，执行必要的版本迁移
3. 执行配置完整性校验（必填字段：`project.name`、`larkResources.baseAppToken`、`larkResources.baseTableIds.todos`）

### 第1步：确定查询主体

**确定 "当前用户"：**
1. 若指定 `--member <姓名>` → 按成员名称解析规则（见 CLAUDE.md）找到 openId
2. 若无 `--member` → 通过 `lark-contact` 获取当前登录用户的 openId（`lark-cli contact +me`）
3. 若 openId 无法确定 → 提示「无法获取当前用户信息，请使用 --member <姓名> 指定」

**`--team` 模式：** 遍历 `config.team.members`，逐人生成速报（串行执行，最终合并展示）。

### 第2步：查询 Base 数据

并行查询当前用户负责的：

| 数据 | 查询条件 | 说明 |
|------|----------|------|
| 昨日完成 | 完成日期 = yesterday，负责人 = 当前用户 | 展示昨天做了什么 |
| 今日计划 | 截止日期 = today，状态 ∈ {待处理,进行中}，负责人 = 当前用户 | 展示今天要做什么 |
| 进行中待办 | 状态 = 进行中，负责人 = 当前用户（截止日期不限于今天）| 补充展示正在推进的事项 |
| 阻塞项 | 截止日期 < today - 3天，状态 ∈ {待处理,进行中}，负责人 = 当前用户 | 逾期超 3 天视为阻塞 |

> 📅 yesterday = currentDate - 1天，详见 [`_shared/date-calc-rules.md`](../_shared/date-calc-rules.md)

**查询容错：** Base 查询失败时展示 `⚠️ 数据查询失败（{错误}）`，继续展示可获取的部分。

### 第3步：生成速报文本

**个人速报格式（`pmo-standup` 默认）：**

```
📌 {项目名} 站会速报 · {today}

✅ 昨日完成（{N} 项）:
  · {待办内容}（{TODO-ID}）
  · {待办内容}（{TODO-ID}）

📋 今日计划（{N} 项）:
  · {待办内容} | {优先级}（{TODO-ID}）
  · {待办内容} | {优先级}（{TODO-ID}）

🔄 进行中（{N} 项）:
  · {待办内容}（{TODO-ID}）

🚧 阻塞项（{N} 项）:
  · {待办内容} — 逾期 {N} 天（{TODO-ID}）
```

**无阻塞项时：**
```
🚧 阻塞项: 无
```

**昨日无完成时：**
```
✅ 昨日完成: 无（今日重点推进计划中事项）
```

**今日无计划时：**
```
📋 今日计划: 无截止今日的待办
  → 进行中：{N} 项（详见上方）
```

**`--team` 全员速报格式（PM 站会主持用）：**

```
📌 {项目名} 全员站会速报 · {today}
════════════════════════════════

👤 张三
  ✅ 昨日完成: {N} 项 · 今日计划: {N} 项 · 🚧 阻塞: {N} 项
  昨日: · {内容}
  今日: · {内容}

👤 李四
  ✅ 昨日完成: 0 项 · 今日计划: {N} 项 · 🚧 阻塞: 0 项
  今日: · {内容}

👤 王五（今日无待办）
  ✅ 昨日完成: 0 项 · 今日计划: 0 项 · 🚧 阻塞: 0 项

────────────────────────────────
汇总：完成 {N} 项 | 计划 {N} 项 | 阻塞 {N} 项
```

### 第4步：推送（--send 时）

> 📋 卡片消息构造详见 [`_shared/feishu-card-template.md`](../_shared/feishu-card-template.md)。

1. 在终端展示速报预览
2. 确认或直接推送（`--send` 不再询问确认，直接推送）：
   - 推送到 `config.larkResources.chatIds` 的第一个群
   - `--team` 模式推送全员速报
3. 推送失败时提示 `⚠️ 推送失败，速报已在终端展示`

**`--send` 时的推送规则：**
- 个人速报：直接发送文本（不 @ 全员）
- `--team` 全员速报：发送文本，阻塞项中 @对应负责人

## 异常处理

| 场景 | 处理方式 |
|------|------|
| 无当前项目 | 提示「请先执行 pmo-use <项目名>」 |
| 无法获取当前用户 openId | 提示「请使用 --member <姓名> 指定」 |
| --member 指定的成员不在项目中 | 提示「{姓名} 不在项目成员列表中，是否仍继续查询？ [y/N]」 |
| Base 查询超时/失败 | 展示可获取的部分，标注 ⚠️ 数据不完整 |
| 推送失败 | 提示 ⚠️，不影响终端展示 |
| --all 中某项目查询失败 | 标注 ⚠️，跳过该项目 |

## 边缘情况

| 场景 | 处理方式 |
|------|------|
| 昨日无完成、今日无计划、无阻塞 | 展示「今日无待办事项，请确认是否有未录入的工作」|
| 进行中待办超过 10 条 | 仅展示截止日期最近的 5 条，标注「共 {N} 条，显示最近 5 条」|
| --all 时关注项目列表为空 | 展示所有 active 项目的汇总 |
| 周一执行（昨日为周末）| 展示上周五的完成情况，标注「上周五完成」|
