# pmo-weekly-digest — 轻量周报推送

> 与 `pmo-weekly-report` 互补。`pmo-weekly-digest` 不生成文档、不归档，直接推送一条消息卡片到项目群，适合每周五下班前快速同步。

**与 pmo-weekly-report 的定位对比：**

| 维度 | pmo-weekly-report | pmo-weekly-digest |
|------|-------------------|-------------------|
| 输出 | 飞书文档 + 知识库归档 | 消息卡片（无文档）|
| 适合场景 | 正式存档、复盘参考 | 快速同步、随手推送 |
| 耗时 | 较长（需生成文档）| 快（仅查询 + 推送）|
| 历史可查 | ✅ 知识库永久保存 | ✅ 群消息历史 |

## 执行方式

```bash
# 生成本周摘要并预览（不推送）
claude pmo-weekly-digest

# 生成并推送到项目群
claude pmo-weekly-digest --send

# 跨所有关注项目，分别推送
claude pmo-weekly-digest --all --send

# 指定推送到某个群（不限于项目群）
claude pmo-weekly-digest --send --chat-id <chat_id>

# 预览（不推送）
claude pmo-weekly-digest --dry-run

# 生成上周的摘要
claude pmo-weekly-digest --week YYYY-MM-DD
```

## 前置条件

已通过 `pmo-use` 设置当前项目（`--all` 模式不需要）。

**Base 读操作超时遵循 CLAUDE.md 公共配置（单次 20s，并发 30s）。所有飞书 API 写操作（IM 推送）遵循公共错误重试策略：3 次指数退避重试（1s/3s/5s）。**

## 执行流程

### 公共：待处理队列检查

执行前检查待处理队列（仅简短提示，不阻塞主流程），详见 [`_shared/pending-queue-check.md`](../_shared/pending-queue-check.md)。

### 配置加载

1. 按 CLAUDE.md「读取当前项目配置」规则加载项目配置
2. 检查 `schemaVersion`，执行必要的版本迁移
3. 执行配置完整性校验（必填字段：`project.name`、`larkResources.baseAppToken`、`larkResources.baseTableIds.*`）

### 第1步：确定周范围

- 默认：本周周一 ~ today（ISO 周，周一为第一天）
- `--week YYYY-MM-DD`：以该日期所在周的周一 ~ 周日为范围

**周编号计算**：按 ISO 8601 标准，如 `第25周（06-09 ~ 06-15）`。

### 第2步：并行查询数据

对每个目标项目，**并行**从 Base 查询本周数据：

| 数据项 | 查询条件 |
|--------|----------|
| 本周会议数 | 会议日期在本周范围内 |
| 本周新增待办数 | 创建时间在本周范围内 |
| 本周完成待办 | 完成日期在本周范围内，最多取前 5 条（按优先级排序）|
| 当前逾期待办数 | 截止日期 < today，状态 ∈ {待处理,进行中} |
| 本周完成里程碑 | 实际完成日期在本周范围内 |
| 下周重点待办 | 截止日期在下周范围，状态 ∈ {待处理,进行中}，取最高优先级前 3 条 |

**查询容错：** 单项查询失败时该指标显示 `-`，不阻塞整体流程。

### 第3步：生成摘要内容

**摘要卡片内容（终端预览格式）：**

```
📊 {项目名} 周报摘要 · 第{N}周（{MM-DD}~{MM-DD}）

会议 {N} 场 | 新增待办 {N} | 完成 {done}（{pct}%）
逾期待办 {N} 条

本周亮点 ✨:
  · {完成待办内容 1}（@{负责人}）
  · {完成待办内容 2}（@{负责人}）
  {若有里程碑完成}· 🏁 {里程碑名称} 完成

下周重点 📌:
  · {截止日期最近的高优待办 1}（@{负责人}，截止 {日期}）
  · {截止日期最近的高优待办 2}（@{负责人}，截止 {日期}）
```

**亮点选取规则：**
- 优先选 P0/P1 已完成待办（最多 3 条）
- 其次本周完成里程碑
- 若无完成记录：展示「本周无完成记录」

**逾期待办为 0 时：** 不展示逾期行，改为展示 `✅ 本周无逾期`

**下周重点为空时（无待办截止于下周）：** 显示「下周计划：待跟进中」

### 第4步：展示预览并推送

1. **终端展示预览**（所有模式都先预览）
2. **推送（--send 时）：**
   - 默认：向 `config.larkResources.chatIds` 的第一个群发送
   - `--chat-id <id>`：向指定群发送
   - `--all --send`：按项目逐一推送（串行，避免频繁打扰）
   - 推送失败时提示 `⚠️ 推送失败（{错误}）`，终端预览仍正常展示

**推送成功后输出：**
```
✅ 摘要已推送
══════════════════════════════
  项目：{项目名}
  周期：第{N}周（{MM-DD}~{MM-DD}）
  目标群：{chat_id}
```

### --dry-run 预览模式

等同于不带 `--send` 执行：只生成预览，在底部标注：
```
── 预览模式 ──
使用 --send 推送到项目群
```

## 异常处理

| 场景 | 处理方式 |
|------|------|
| 无当前项目（非 --all 模式）| 提示「请先执行 pmo-use <项目名>」 |
| 项目无群 chatId | 提示「项目未配置群 chatId，请通过 pmo-use 添加」，展示预览但不推送 |
| Base 查询全部失败 | 提示「数据查询失败，请检查网络/权限」，展示空摘要骨架 |
| 推送失败（lark-im 错误）| 提示 ⚠️ + 错误信息，终端预览仍可看 |
| --all 且无项目 | 提示「无已注册项目，请先运行 pmo-init」 |
| --all 某项目查询失败 | 跳过该项目，汇总中标注 ⚠️ |

## 边缘情况

| 场景 | 处理方式 |
|------|------|
| 本周无任何数据（新项目）| 展示「本周暂无记录，项目刚启动」 |
| 本周完成率 0%（无完成）| 不展示「亮点」，改为展示「本周无完成记录，重点推进：{进行中待办 Top 3}」|
| 完成待办数 > 新增待办数 | 完成率显示「>{N}%（本周消化了历史积压）」 |
| --all 只有一个项目 | 与单项目模式相同 |
| 周五执行（下周重点已知）| 正常展示下周重点待办 |
