# pmo-notify — 主动提醒

> 自动检查待办和里程碑，向项目群推送提醒卡片。适合与飞书定时任务结合使用，实现每日自动提醒。

## 执行方式

```bash
# 向当前项目的项目群推送提醒
claude pmo-notify

# 预览提醒内容，不实际推送
claude pmo-notify --dry-run

# 向指定项目推送（无需切换上下文）
claude pmo-notify --project XRay

# 向所有关注项目推送
claude pmo-notify --all

# 自定义提醒范围（默认：过期待办 + 今天截止 + 7天内里程碑）
claude pmo-notify --overdue-only          # 仅过期待办
claude pmo-notify --milestone-only        # 仅里程碑提醒
```

## 前置条件

- 当前项目或 `--project` 指定的项目已在 Base 中配置
- 项目配置中有 `larkResources.chatIds`（至少一个群）

## 执行流程

### 第1步：确定目标项目

1. 若指定 `--project` → 仅该项目
2. 若指定 `--all` → 遍历 pinned 或所有 active 项目
3. 否则使用当前项目

### 第2步：采集提醒数据

对每个目标项目，并行查询：

| 数据 | 查询条件 | 展示优先级 |
|------|---------|-----------|
| 过期待办 | 截止日期 < currentDate，状态≠已完成/已取消 | 🔴 最高 |
| 今日截止 | 截止日期 = currentDate，状态≠已完成/已取消 | 🟡 高 |
| 明日截止 | 截止日期 = currentDate + 1，状态≠已完成/已取消 | 🟠 中 |
| 本周截止 | 截止日期在 7 天内（不含今天/明天），状态≠已完成/已取消 | 🔵 低 |
| 里程碑到期 | 计划日期在 7 天内，状态≠已完成/已取消 | 🟡 高 |

### 第3步：生成提醒卡片

```
📋 {项目名} 待办提醒 · {today}

🔴 已过期（{N} 项）:
  · {待办内容} — @{负责人}（逾期 {N} 天）
  · {待办内容} — @{负责人}（截止 {日期}）

🟡 今日截止（{N} 项）:
  · {待办内容} — @{负责人}

🏁 里程碑即将到期（{N} 项）:
  · {里程碑名称} — 计划 {日期}（剩余 {N} 天）

───────────────
📊 Base 直达 → {baseUrl}
```

### 第4步：推送

- `--dry-run`：仅在终端展示卡片，不推送
- 默认：推送到项目配置 `larkResources.chatIds` 的第一个群
- `--all`：按项目逐一推送（串行，避免频繁 @全员）
- 若该项目无提醒内容 → 不推送（安静模式）

**推送规则：**
- 过期待办 > 0 条时，在卡片首行 @负责人
- 仅里程碑到期时减少 @（非紧急）
- 推送间隔 ≥ 30 分钟，避免同一项目重复打扰

## 异常处理

| 场景 | 处理 |
|------|------|
| 项目无群 chatId | 仅在终端展示，提示「未配置项目群，无法推送」 |
| 推送失败 | 提示 ⚠️ + 终端展示完整提醒内容 |
| Base 查询失败 | 提示 ⚠️ 该项目数据获取失败 |
| 无任何待提醒项 | 展示 `✅ {项目名} 一切正常，无需提醒` |

## 定时执行建议

将以下 cron 添加到 Claude Code 实现每日自动提醒：

```bash
# 每个工作日 9:57 执行
claude pmo-notify --all
```

> ⚠️ 定时执行时建议固定 `--dry-run` 先验证，确认无误后再去掉。
