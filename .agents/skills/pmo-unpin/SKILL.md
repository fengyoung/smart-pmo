# pmo-unpin — 取消关注项目

## 前置条件

无。pmo-unpin 仅操作本地文件 `~/.smart-pmo/pinned`，不依赖飞书 API。

## 执行方式

```bash
# 取消关注一个或多个项目
claude pmo-unpin <项目名> [<项目名>...]
```

## 执行流程

> 本 Skill 仅执行纯本地文件操作，无需检查待处理队列或调用飞书 API。

### 取消关注

1. 解析命令行参数，获取要取消关注的项目名列表（支持空格分隔多个项目名）
2. 若未传项目名 → 显示当前关注列表 + 使用提示：
   ```
   📌 当前关注项目（{N} 个）：XRay, RCA
   
   使用 pmo-unpin <项目名> 取消关注
   ```
3. 读取 `~/.smart-pmo/pinned` 文件：
   - 文件不存在 → 提示「当前无关注项目」，结束
4. 从列表中移除指定项目名：
   - 项目名存在于 pinned → 移除
   - 项目名不存在于 pinned → 静默跳过
5. 写回文件（若全部移除，写入空文件）
6. 输出确认

**输出格式：**

```
📌 已取消关注
═══════════════════════════════
  ❌ XRay — 已移除
───────────────────────────────
当前关注 {N} 个项目：RCA

使用 pmo-pin <项目名> 可重新关注
```

**移除多项时：**

```
📌 已取消关注
═══════════════════════════════
  ❌ XRay — 已移除
  ❌ RCA — 已移除
───────────────────────────────
当前无关注项目

pmo-dashboard 将展示所有 active 项目
使用 pmo-pin <项目名> 关注项目
```

## 存储格式

`~/.smart-pmo/pinned` 为纯文本，每行一个项目名（project_id = registry 文件名不含 `.json`）。

## 边缘情况

| 场景 | 处理方式 |
|------|---------|
| 未传项目名 | 显示当前关注列表 + 使用提示 |
| pinned 文件不存在 | 提示「当前无关注项目」，结束 |
| 项目名不在 pinned 中 | 静默跳过，输出中不展示 |
| 全部移除后为空 | 提示「当前无关注项目，pmo-dashboard 将展示所有 active 项目」 |
