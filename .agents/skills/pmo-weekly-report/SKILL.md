# pmo-weekly-report — 周报生成

## 执行方式

```bash
# 生成本周周报
claude pmo-weekly-report

# 生成指定周的周报（传入该周任意一天，自动计算周一到周日范围）
claude pmo-weekly-report --week YYYY-MM-DD

# 预览周报草稿，不写入（dry-run）
claude pmo-weekly-report --dry-run

# 不对比上周数据
claude pmo-weekly-report --no-compare

# 生成并推送摘要卡片到项目群
claude pmo-weekly-report --send

# 生成并推送到指定群（不推送到项目群）
claude pmo-weekly-report --send --chat-id <chat_id>
```

## 前置条件

已通过 `pmo-use` 设置当前项目。

**所有飞书 API 写操作（Wiki 归档、文档创建、消息推送）遵循公共错误重试策略（见 CLAUDE.md）：3 次指数退避重试（1s/3s/5s）。**

## 执行流程

### 公共：待处理队列检查

> 📋 详见 [`_shared/pending-queue-check.md`](../_shared/pending-queue-check.md)。执行开始时检查 `~/.smart-pmo/` 下的四个待处理目录。
>
> 📋 卡片消息构造（`--send` 模式）详见 [`_shared/feishu-card-template.md`](../_shared/feishu-card-template.md)。

### 配置加载

1. 按 CLAUDE.md「读取当前项目配置」规则加载项目配置
2. 检查 `schemaVersion`，执行必要的版本迁移
3. 执行配置完整性校验（必填字段：`project.name`、`larkResources.baseAppToken`、`larkResources.baseTableIds.*`、`larkResources.wikiSpaceId`、`larkResources.wikiNodeTokens.02-周报`）

### 1. 确定周范围

- 默认：本周周一 ~ 本周周日（ISO 周，周一为第一天）
- `--week YYYY-MM-DD`：以该日期所在周的周一 ~ 周日为范围
- 同时计算上周范围（用于历史对比）

**周编号计算**：按 ISO 8601 标准，如 `第24周（06-08 ~ 06-14）`。

### 2. 去重检查（已生成过本周周报）

在拉取数据前，先检查是否已生成过本周周报：

1. 通过 `lark-wiki` 查询 `config.larkResources.wikiNodeTokens["02-周报"]` 对应目录下的文档列表
2. 匹配文件名模式 `{YYYYMMDD}-*`，查找是否存在日期在本周范围内的文档
3. 若存在匹配文档：
   ```
   ⚠️ 本周周报可能已存在：
     · 20260612-第24周周报（创建于 06-12）

   是否继续？[y/N]
   ```
   - `y` → 继续生成，新周报将使用 **不同的文档标题**（追加 `-2`、`-3` 等序号），而非覆盖旧文档
     - 例：已存在 `20260612-第24周周报` → 新文档命名为 `20260612-第24周周报-2`
     - 序号从 2 开始，依次递增直至无冲突
   - `N` 或直接回车 → 取消操作

### 3. 并行数据收集

**并行**从 Base 收集本周和上周数据：

**本周数据：**
- 本周会议记录（会议日期在本周范围内）
- 本周新增待办（创建时间在本周范围内）
- 本周完成待办（完成日期在本周范围内）
- 当前逾期待办（截止日期 < currentDate 且 状态≠已完成/已取消）
- 里程碑状态快照（所有状态≠已取消的里程碑）
- 下周计划：截止日期在下周一～下周日之间、状态≠已完成/已取消的待办，按优先级排序

**上周数据（用于对比，若 `--no-compare` 则跳过）：**
- 上周完成待办数（完成日期在上周范围内）
- 上周新增待办数
- 上周末时的逾期待办数（**估算值**：截止日期 < 上周日 且 在上周日时状态≠已完成；注：无历史快照，实际值可能偏差）
- 里程碑进度快照（通过会议索引或里程碑历史推算，若无历史数据则标注"无上周数据"）

**数据收集容错：**
- Base 查询超时（见 CLAUDE.md 超时配置，默认 30s）→ 对应指标标注 `-`，不阻塞整体流程
- 单表查询失败 → 对应部分在周报中显示「数据暂不可用」

### 4. 计算对比指标

- 待办完成率变化：本周完成数/本周新增数 vs 上周完成数/上周新增数
- 逾期待办变化：+N（增加）/ -N（减少）/ 持平
- 本周已完成里程碑列表
- 估算数据（如逾期趋势）标注 `*估算`

### 5. AI 整理周报内容

**周报结构：**

```
# {项目名} 周报 · {周期}

## 📊 本周概览
| 维度 | 本周 | 上周 | 变化 |
|------|------|------|------|
| 本周会议 | {N} 次 | {N} 次 | {↑/↓/持平} |
| 新增待办 | {N} 项 | {N} 项 | {↑N/↓N/持平} |
| 完成待办 | {N} 项 | {N} 项 | {↑N/↓N/持平} |
| 完成率 | {X}% | {X}% | {↑/↓ △%} |
| 逾期待办 | {N} 项 | {N} 项 | {↑N/↓N/持平} |
| 里程碑进行中 | {N} 个 | {N} 个 | — |

## 📝 本周会议

| 日期 | 主题 | 产出待办 | 关键决策 |
|------|------|---------|---------|
| {MM-DD} | {会议主题} | {N} 项 | {决策摘要} |

（无会议时显示「本周无会议记录」）

## ✅ 完成事项

### 本周完成
- {TODO-xxx} {内容} — {负责人}（{完成日期}）

### ⚠️ 已逾期
- {TODO-xxx} {内容} — {负责人}（逾期 {N} 天）

（无已逾期时显示「✅ 无逾期待办」）

## 🏁 里程碑进展

| 里程碑 | 计划日期 | 状态 | 进度 | 备注 |
|--------|---------|------|------|------|
| {名称} | {日期} | ✅ 已完成 | 100% | 本周完成 |
| {名称} | {日期} | 🔄 进行中 | {N}% | |
| {名称} | {日期} | ⏳ 未开始 | 0% | |

## 📌 下周计划

1. [{TODO-xxx}] {计划事项} — @{负责人}（{截止日期}）
2. ...

## 💬 风险与问题

| 风险/问题 | 影响 | 应对措施 |
|-----------|------|---------|
| {描述} | {影响范围} | {措施} |

（AI 根据逾期待办和延期里程碑自动识别风险项；无风险时显示「无显著风险」）

---

> 本报告由 Smart-PMO 自动生成
> 数据来源：PMO 管理台 Base
```

### 6. 展示与确认

展示草稿给用户确认：

- 对比数据用 `↑3` / `↓2` / `持平` 标注
- 逾期趋势等估算数据附注 `*估算`
- 用户可选择：
  - **确认写入** — 继续步骤 7
  - **修改** — 用户输入修改意见，AI 调整后重新展示
  - **取消** — 丢弃草稿

### 7. 归档与推送

用户确认后：

1. **创建飞书文档**：通过 `lark-doc` 创建周报文档
2. **归档到知识库**：通过 `lark-wiki` 将文档归档到 `config.larkResources.wikiNodeTokens["02-周报"]` 对应的目录节点
   - 文档标题格式 `{YYYYMMDD}-{标题}`（如 `20260612-第24周周报`）
   - 遵循知识库文档命名格式约定
3. **推送摘要**（若指定 `--send`）：

   通过 `lark-im` 向目标群发送消息卡片：

   ```
   📊 {项目名} 周报 · {周期}
   
   会议：{N} 场 | 新增待办：{N} | 完成：{N}（{X}%）
   逾期待办：{N}（↑/↓ {△} vs 上周）
   里程碑：{N} 项进行中，{N} 项本周完成
   
   📎 查看完整周报 → {wiki文档链接}
   ```

   **目标群规则：**
   - `--send`：推送到项目配置 `larkResources.chatIds` 的第一个群
   - `--send --chat-id <id>`：推送到指定群（不限于项目群）
   - **推送前预览**：`--send` 时先在终端渲染卡片预览，用户确认后再推送
   - 推送失败时提示 `⚠️ 消息推送失败，周报已归档`，不阻塞主流程

4. 输出确认：

```
✅ 周报已生成
═══════════════════════════════
  文档：20260612-第24周周报
  归档：02-周报/
  链接：{wiki文档链接}
  {推送状态}
```

### --dry-run 预览模式

```
claude pmo-weekly-report --dry-run
```

仅执行步骤 1-5（确定周范围 → 去重检查 → 数据收集 → 计算指标 → AI 整理），展示完整草稿后结束，**不创建文档、不归档、不推送**。

底部显示提示：

```
── 预览模式，未写入 ──
使用 pmo-weekly-report 正式生成
使用 pmo-weekly-report --send 生成并推送
```

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 本周无会议 | 周报正常生成，会议部分显示「本周无会议记录」 |
| 本周无新增/完成待办 | 显示 0，不影响周报生成 |
| 无上周数据可对比 | 对比段落显示「首次生成，暂无上周数据」 |
| 已生成过本周周报 | 提示已存在，询问是否继续 |
| Base 查询超时/失败 | 对应指标显示 `-`，不阻塞整体流程 |
| Wiki 归档失败（重试耗尽） | 提示「❌ 归档失败，请手动将文档移至 02-周报/」 |
| --send 推送失败 | 提示 ⚠️ 推送失败，周报已归档，不阻塞主流程 |
| 无当前项目 | 提示「请先执行 pmo-use <项目名>」 |
| wikiNodeTokens 缺少 02-周报 | 提示「⚠️ 缺少 02-周报 目录节点，请检查项目配置」 |
