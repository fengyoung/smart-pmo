# pmo-list — 列出所有项目

## 执行方式

```bash
# 列出所有项目（含实时统计）
claude pmo-list

# 仅列出 active 项目
claude pmo-list --active

# 仅列出配置信息（不查询 Base，快速模式）
claude pmo-list --config-only

# JSON 格式输出（适用于脚本/管道消费）
claude pmo-list --json
```

## 执行流程

1. 遍历 `~/.smart-pmo/registry/*.json`
2. 读取每个项目的：`name`、`alias`、`status`、`pm.name`、`schemaVersion`
3. 读取 `~/.smart-pmo/current` 标记当前项目
4. 读取 `~/.smart-pmo/pinned` 标记关注项目

5. **并行**查询每个 active 项目的 Base 实时统计（`--config-only` 模式跳过此步）：
   - 待办总数 / 待处理数 / 已过期数
   - 里程碑总数 / 进行中数 / 即将到期数（7天内）
   - 查询超时 10s 的项目标注 `⚠️ 超时`，不阻塞其他项目

6. 展示表格：

```
所有项目（{N} 个）：

  ★ 当前项目 → {项目名} ({代号})  ※ 关注

  ┌───────────────┬──────┬────────┬──────────┬──────────────┬──────────────┐
  │ 项目名称       │ 代号 │ 状态   │ 项目经理 │ 待办（待/过期）│ 里程碑（进/将）│
  ├───────────────┼──────┼────────┼──────────┼──────────────┼──────────────┤
  │ {项目名}      │ XRay │ active │ 邹晨风   │ 12 (5/2‼️)   │ 4 (2/1⚠️)    │
  │ {项目名}      │ ICS  │ active │ 张三     │ 8 (3/0)      │ 3 (1/0)      │
  │ {项目名}      │ PMS  │ archived│ 李四   │ -            │ -            │
  └───────────────┴──────┴────────┴──────────┴──────────────┴──────────────┘

  注：待（待处理数）/ 过期（已过期数‼️）  |  进（进行中数）/ 将（7天内到期⚠️）
  archived 项目不查询 Base 实时数据

  使用 pmo-use <项目名> 切换项目
  使用 pmo-pin <项目名>  关注项目
  使用 pmo-dashboard     查看多项目概览
```

**Base 查询失败的优雅降级：**
- 单个项目超时/失败 → 该行统计显示 `⚠️`，其他项目正常展示
- 全部项目查询失败 → 回退到 `--config-only` 模式，仅展示配置信息
