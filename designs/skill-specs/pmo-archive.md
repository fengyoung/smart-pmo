# pmo-archive — 文档归档 Skill 设计

> P0 优先级 | 文件归档到知识库

---

## 调用方式

```bash
# 归档文件到指定目录
claude pmo-archive --file <本地文件路径> [--dir <目录名>]

# 示例
claude pmo-archive --file /path/to/需求文档v2.docx --dir 03-需求文档
claude pmo-archive --file /path/to/UI设计稿.png --dir 05-项目资料
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `--file` | string | ✅ | 本地文件路径 |
| `--dir` | string | 可选 | 目标目录名，不指定则交互选择 |
| `--rename` | string | 可选 | 指定文件名（不指定则用原始文件名）|

---

## 执行流程

```
用户输入: claude pmo-archive --file ./需求文档v2.docx
         │
         ▼
① 读取文件
   ├→ 检查文件是否存在
   ├→ 检查文件类型（支持：docx/pdf/xlsx/pptx/png/jpg/md/txt）
   └→ 文件大小限制：≤ 50MB
         │
         ▼
② 如果不带 --dir 参数，交互选择目录
   ┌─────────────────────────────────────┐
   │ 请选择归档目录：                      │
   │                                     │
   │ 1. 01-会议纪要（当前：3 个文件）      │
   │ 2. 02-周报（当前：1 个文件）          │
   │ 3. 03-需求文档 ← 推荐                │
   │ 4. 04-设计文档                       │
   │ 5. 05-项目资料                       │
   │ 6. 99-归档                           │
   │                                     │
   │ 请输入编号或目录名: _3_               │
   └─────────────────────────────────────┘
         │
         ▼
③ 通过 lark-wiki 上传文件到知识空间
   ├→ API: upload_media → 到知识库指定目录节点
   ├→ 文件自动重命名规则：
   │   └→ 不指定 --rename → 使用原始文件名
   │   └→ 如果文件名不含日期 → 自动添加日期前缀
   └→ 记录文档链接
         │
         ▼
④ 归档完成确认
   ✅ 已归档: 需求文档v2.docx
      📎 [查看文件]  ← 知识库链接
      目录: 03-需求文档
      日期: 2026-06-09
```

---

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 文件不存在 | 提示检查路径 |
| 文件类型不支持 | 提示支持的格式列表 |
| 文件过大（>50MB）| 建议压缩后上传 |
| 知识库上传失败 | 提示手动上传，显示目标目录 |
| 目录不存在 | 提示可用目录列表 |
