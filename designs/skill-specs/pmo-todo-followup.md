# pmo-todo-followup — 待办跟进 Skill 设计

> P0 优先级 | 查看、跟进、关闭待办

---

## 调用方式

```bash
# 查看所有待办
claude pmo-todo-followup

# 指定筛选条件
claude pmo-todo-followup --mine          # 只看我的待办
claude pmo-todo-followup --overdue       # 只看已过期
claude pmo-todo-followup --status 待处理   # 按状态筛选

# 操作待办
claude pmo-todo-followup --complete TODO-003    # 标记完成
claude pmo-todo-followup --modify TODO-003 --assign @李四  # 转交
claude pmo-todo-followup --modify TODO-003 --due 2026-06-20  # 改截止日期
```

---

## 执行流程

### 查看待办

```
① 从 Base「待办事项」表读取所有记录
   └→ 过滤条件取决于参数（--mine / --overdue / --status）
② 计算统计信息
③ 展示结果：

   📋 待办列表 — 智能客服平台 (8 项)
   ═══════════════════════════════════
   ⚠️ 已过期（2 项）:
   ┌──────┬────────────────────┬─────────┬──────────┐
   │ ID   │ 内容                │ 负责人  │ 截止日期  │
   ├──────┼────────────────────┼─────────┼──────────┤
   │ 003  │ 确定UI方案          │ 李四    │ 06-06 ‼️ │
   │ 005  │ 后端接口联调        │ 王五    │ 06-08 ‼️ │
   └──────┴────────────────────┴─────────┴──────────┘

   待处理（5 项）:
   ┌──────┬────────────────────┬─────────┬──────────┬────────┐
   │ ID   │ 内容                │ 负责人  │ 截止日期  │ 优先级 │
   ├──────┼────────────────────┼─────────┼──────────┼────────┤
   │ 001  │ Sprint评审准备     │ 张三    │ 06-12    │ P0     │
   │ 002  │ 更新项目计划       │ 张三    │ 06-11    │ P1     │
   │ 004  │ 情感分析需求文档   │ 王五    │ 06-20    │ P2     │
   │ 006  │ V2.0接口联调       │ 李四    │ 06-16    │ P1     │
   │ 007  │ 准备演示环境       │ 张三    │ (未指定) │ P2     │
   └──────┴────────────────────┴─────────┴──────────┴────────┘

   操作: 使用参数快速操作:
   pmo-todo-followup --complete <ID>    # 标记完成
   pmo-todo-followup --modify <ID> ...  # 修改内容
```

### 标记完成

```
① 通过 lark-base update_record 更新「待办事项」表
   ├→ 状态 = "已完成"
   └→ 完成日期 = 当天
② 展示确认：
   ✅ TODO-003 已标记为完成
③ 可选：推送到项目群
   ✅ 李四已完成: 确定UI方案
```

### 修改待办

```
① 展示当前待办信息
② 交互式引导修改
③ 通过 lark-base update_record 更新
④ 展示确认
```

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 待办列表为空 | 展示"暂无待办，继续保持！" |
| 待办 ID 不存在 | 提示"未找到该待办，请检查 ID" |
| Base 连接失败 | 提示稍后重试 |
| 过期数量较多（>10）| 按负责人分组展示，不逐个列出 |
