# pmo-init — 项目初始化 Skill 设计

> P0 优先级 | 交互式创建新项目

---

## 功能概述

交互式创建新项目，自动完成全部基础设施搭建并注册到 `~/.smart-pmo/registry/`。

---

## 调用方式

```bash
claude pmo-init
# 交互式执行，按提示输入信息
```

---

## 执行流程

### Step 1：交互收集项目信息

```
pmo-init
┌─────────────────────────────────────────┐
│ 📋 新项目初始化                          │
│                                         │
│ 请输入项目名称: _智能客服平台_            │
│ 请输入项目代号（可选）: _ICS_             │
│                                         │
│ ─── 团队信息 ───                         │
│ 项目经理姓名: _张三_                     │
│ 请选择项目经理的飞书账号（搜索）: 张三    │
│                                         │
│ 是否添加核心成员? [Y/n] _Y_              │
│ 成员姓名: _李四_   角色: _开发_          │
│ 成员姓名: _王五_   角色: _产品_          │
│ ...                                      │
│                                         │
│ ─── 飞书资源关联 ───                     │
│ 项目群名称/ID: _ICS 项目群_              │
│ 搜索确认: oc_xxxxxxxxxxxx                │
│                                         │
└─────────────────────────────────────────┘
```

### Step 2：创建飞书知识空间

```
调用 lark-wiki 创建空间
├→ 空间名称: [项目名] 知识空间
├→ 空间描述: [项目名] 的项目文档归档空间
└→ 创建目录结构:
   ├── 01-会议纪要/
   ├── 02-周报/
   ├── 03-需求文档/
   ├── 04-设计文档/
   ├── 05-项目资料/
   └── 99-归档/
```

### Step 3：创建多维表格 Base

```
调用 lark-base 创建 Base
├→ Base 名称: [项目名]-PMO-管理台
├→ 创建表1: 待办事项（按 designs/base-tables.md 字段定义）
├→ 创建表2: 里程碑（同上）
├→ 创建表3: 会议记录索引（同上）
└→ 建立表间关联（待办事项 → 会议索引、里程碑 → 待办事项）
```

### Step 4：生成配置文件

```
├→ 写入 ~/.smart-pmo/registry/<项目名>.json
├→ 写入 ~/.smart-pmo/current → 指向新项目
└→ 如果 team.members 数量≥2，自动 pmo-pin 该项目
```

### Step 6：输出配置摘要

```
初始化完成后，输出当前项目配置摘要：

┌─────────────────────────────────────┐
│ 📋 当前项目: 智能客服平台 (ICS)      │
│                                     │
│ 知识空间:   ss_abc123                │
│ 管理台Base: bas_abc123               │
│ 项目群:     oc_chat_001              │

│                                     │
│ 使用 pmo-use <项目名> 切换项目       │
│ 使用 pmo-list        查看所有项目    │
└─────────────────────────────────────┘
```

---

## 交互确认点

| 步骤 | 是否需要用户确认 | 说明 |
|------|----------------|------|
| 收集完信息后 | ✅ | 显示所有信息摘要，确认后再执行创建 |
| 创建知识空间前 | ✅ | 确认空间名称 |
| 创建 Base 前 | ✅ | 确认 Base 名称和表结构 |
| 写入注册表时 | ❌ | 成功率>99%，无需确认 |

---

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 知识空间创建失败 | 提示用户手动创建，录入 spaceId 继续 |
| Base 创建失败 | 提示用户手动创建，录入 appToken |
| 项目名已存在 | 提示"项目已注册，是否覆盖? [y/N]" |
| 飞书 API 频率限制 | 自动重试 3 次，每次等待 1s |
| 用户中途退出 (Ctrl+C) | 已创建的资源不清理，提示"可重新执行 pmo-init 或手动清理" |
