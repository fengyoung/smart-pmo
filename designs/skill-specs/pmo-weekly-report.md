# pmo-weekly-report — 周报生成 Skill 设计

> P1 优先级 | 自动汇总项目周度进展

---

## 调用方式

```bash
# 生成本周周报
claude pmo-weekly-report

# 生成指定周
claude pmo-weekly-report --week 2026-06-01  # 从该日期开始的周
```

---

## 执行流程

```
① 确定周范围：
   └→ 本周一 00:00 ~ 本周日 23:59（或 --week 指定）
         │
         ▼
② 从 Base 收集数据
   ├→ 「会议记录索引」表：本周内的会议（主题、日期、决策数）
   ├→ 「待办事项」表：
   │   ├→ 本周新增的待办数
   │   ├→ 本周完成的待办数
   │   ├→ 截止线在本周的待办数
   │   └→ 逾期未完成数（截止日期 < 今天且状态≠已完成/已取消）
   └→ 「里程碑」表：里程碑状态快照
         │
         ▼
③ AI 整理周报内容
   ├→ 本周重点事项摘要
   ├→ 待办完成率分析
   ├→ 里程碑进展
   └→ 下周计划（从待办中提取）
         │
         ▼
④ 展示周报草稿给用户确认
         │
         ▼
⑤ 用户确认后：
   ├→ 通过 lark-doc 创建格式化周报文档
   ├→ 通过 lark-wiki 归档到 02-周报/
   └→ 推送周报摘要到项目群
```


## 推送格式

写入完成后推送到项目群（卡片消息）：

```
📊 [项目名] 第X周周报已发布
═══════════════════════════════
📋 本周完成：3 | 新增：5 | 逾期：2
🏁 里程碑：2/4 进行中
📝 会议：3 次

📎 [查看完整周报]
```

---

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 本周无会议 | 周报正常生成，会议部分显示"本周无会议" |
| 本周无新增/完成待办 | 显示 0，不影响周报生成 |
| 周报内容较多 | 自动分节，推送摘要卡片到群 |
| 已生成过本周周报 | 提示"本周周报已存在，是否覆盖？[y/N]" |
