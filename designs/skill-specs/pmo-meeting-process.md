# pmo-meeting-process — 会议处理 Skill 设计

> P0 优先级 | 核心模块，三条输入路径

---

## 功能概述

处理会议录音/转写内容，自动提取纪要、关键决策和待办事项，归档到知识库并写入 Base。

---

## 调用方式

```bash
# 方式A：飞书妙记
claude pmo-meeting-process --minutes <妙记链接或ID>

# 方式B：外部转写文件
claude pmo-meeting-process --file <本地文件路径>

# 方式C：直接粘贴/输入会议记录文本
claude pmo-meeting-process --text

# 手动指定主题/日期（可选）
claude pmo-meeting-process --minutes <链接> --topic "Sprint评审" --date 2026-06-09
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `--minutes` | string | 三选一 | 飞书妙记链接或 session ID |
| `--file` | string | 三选一 | 本地转写文件路径（.txt / .docx / .md）|
| `--text` | flag | 三选一 | 提示用户粘贴或直接输入会议记录文本 |
| `--topic` | string | 可选 | 手动指定会议主题（不指定则 AI 自动识别）|
| `--date` | string | 可选 | 手动指定会议日期 YYYY-MM-DD（不指定则自动识别）|

---

## 前置条件

1. 已通过 `pmo-use` 设置当前项目
2. 对于妙记方式：用户有妙记访问权限
3. 对于文件方式：文件存在且格式正确

---

## 执行流程

### 第1步：获取当前项目配置

```
读取 ~/.smart-pmo/current → 加载 ~/.smart-pmo/registry/<项目名>.json
如果未找到配置 → 提示"请先执行 pmo-use <项目名> 设置当前项目"
```

### 第2步：读取会议内容

**输入源A — 飞书妙记：**
```
① 解析妙记链接，提取 session ID
② 通过 lark-minutes 获取转写内容
③ 获取会议元信息：主题、时间、参会人
```

**输入源B — 外部转写文件：**
```
① 读取本地文件（支持 .txt / .docx / .md）
② 自动检测编码 UTF-8 / GBK
③ 大小限制 ≤ 10MB
④ AI 从内容中识别会议主题、日期、参会人
```

**输入源C — 直接输入文本（--text）：**
```
① 提示用户粘贴或输入会议记录内容
② 以空行+EOF（或用户表示输入完毕）作为结束标记
③ 后续处理与输入源B相同
④ 记录方式字段写入"手动记录"
```

### 第3步：AI 提取结构化信息

**会议元信息提取：**

| 字段 | 提取方式 | 兜底策略 |
|------|---------|---------|
| 会议主题 | 参数指定或 AI 从转写内容识别 | 无法识别时提示用户用 `--topic` 指定 |
| 会议日期 | 参数指定或 AI 从上下文识别 | 无法识别时使用当前日期 |
| 参会人 | 妙记元数据或 AI 从对话推断 | 以用户确认为准 |
| 讨论要点 | AI 分段总结，每段 1-2 句 | — |
| 关键决策 | AI 识别决策点 | — |

**待办事项提取规则：**

从转写内容通过以下语义模式识别待办：

| 模式 | 示例 | 提取 |
|------|------|------|
| "XX需要/要负责/会处理YY" | "李四负责接口联调" | 内容=接口联调，负责人=李四 |
| "我来做YY" | "我来做演示环境" | 内容=准备演示环境，负责人=说话者 |
| "YY要在DDL前完成" | "周五前要完成方案评审" | 截止日期=本周五 |
| "交给XX" | "这个交给王五处理" | 负责人=王五 |
| 明确标记 | "TODO: 内容" | 直接提取 |

**截止时间解析规则：**

| 表达 | 处理 |
|------|------|
| "尽快" | 当天+3天 |
| "这周五/下周X" | 计算具体日期 |
| 明确日期（"6月15日"）| 使用具体日期 |
| 未提及 | **截止日期留空**，确认界面标注 ⚠️ 截止日期待确认 |

**成员名称解析（负责人字段）：**

```
识别到姓名后，按以下顺序解析为 openId：
1. 在项目配置 team.members 列表中按 name 精确匹配
2. 匹配不到 → 通过 lark-contact 搜索飞书通讯录
3. 仍找不到 → 负责人字段留空，确认界面标注 ⚠️ @{姓名} 未匹配
```

### 第4步：展示结果给用户确认

```
📋 会议处理结果 · 请确认
──────────────────────────
会议主题: {主题}
会议日期: {日期}
参会人: {列表}

── 讨论要点 ──
1. {要点1}
2. {要点2}

── 关键决策 ──
• {决策1}
• {决策2}

── 待办事项 ──
□ {内容} | @{负责人} | 截止: {日期}
□ {内容} | @{负责人} | 截止: ⚠️ 待确认（未识别到截止时间）
□ {内容} | @??? | 截止: {日期}  ← ⚠️ 负责人未匹配，请确认

[确认写入] [修改后确认] [取消]
```

等待用户选择后进入下一步。

### 第5步：写入数据

**① 生成会议纪要文档：**
```
- 使用模板创建结构化会议纪要
- 通过 lark-doc 创建飞书文档
- 通过 lark-wiki 归档到 01-会议纪要/{YYYY-MM-DD}-{主题}.md
```

**② 写入 Base 待办事项表：**

对每个待办，通过 `lark-base` 写入以下字段：

| 字段 | 值 |
|------|-----|
| 待办内容 | 提取内容 |
| 负责人 | 解析后的 openId（可为空） |
| 截止日期 | 解析后的日期（可为空） |
| 状态 | 待处理 |
| 优先级 | P2-一般 |
| 来源 | "会议:{会议主题}" |
| 所属会议 | 关联本次会议记录 |

去重：写入前检查是否已在 Base 中（按内容语义相似度 + 同会议来源），相似度 >80% 则跳过。

**③ 写入 Base 会议记录索引表：**

通过 `lark-base` 写入以下字段：

| 字段 | 值 |
|------|-----|
| 会议主题 | 提取/指定值 |
| 会议日期 | 提取/指定值 |
| 开始/结束时间 | 从妙记获取或留空 |
| 参会人 | 解析后的列表 |
| 记录方式 | 妙记/文件/手动记录 |
| 来源文件 | 妙记链接或本地路径 |
| 纪要文档链接 | 刚创建的文档链接 |
| 讨论要点摘要 | AI 提取内容 |
| 关键决策 | AI 提取内容 |
| 产出待办 | 关联上一步写入的待办记录 |

### 第6步：推送到项目群

通过 `lark-im` 发送卡片消息到配置的 `chatId`：

```
📋 新会议纪要已归档

会议：{主题} · {日期}
产出待办：{N} 项

关键决策：
• {决策1}

📎 [查看完整纪要]
```

---

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 无当前项目 | 提示运行 pmo-use |
| 妙记不存在或无权限 | 提示并建议使用 `--file` 方式 |
| 文件格式不支持 | 提示支持的格式（txt/docx/md）|
| 文件过大（>10MB） | 建议拆分或提取关键段落 |
| AI 未能提取到待办 | 提示"未识别到待办，是否仍写入会议纪要？[y/N]" |
| Base 写入失败 | 纪要文档继续归档，待办暂存提示用户手动补录 |
| 知识库归档失败 | 提示用户手动上传，提供文档内容 |
