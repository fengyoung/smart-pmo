# pmo-meeting-process — 会议处理 Skill 设计

> P0 优先级 | 核心模块，两条输入路径

---

## 功能概述

处理会议录音/转写内容，自动提取纪要、关键决策和待办事项，归档到知识库并写入 Base。

---

## 调用方式

```bash
# 方式A：飞书妙记
claude pmo-meeting-process --minutes <妙记链接或ID>

# 方式B：外部转写文件  
claude pmo-meeting-process --file <本地文件路径>

# 方式C：查看帮助
claude pmo-meeting-process --help
```

### 参数说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `--minutes` | string | 二选一 | 飞书妙记链接或 session ID |
| `--file` | string | 二选一 | 本地转写文件路径（.txt / .docx / .md）|
| `--topic` | string | 可选 | 手动指定会议主题（不指定则 AI 自动识别）|
| `--date` | string | 可选 | 手动指定会议日期 YYYY-MM-DD（不指定则自动识别）|

## 执行流程

### 输入源A：飞书妙记

```
用户输入: claude pmo-meeting-process --minutes https://meetings.feishu.cn/xxx
         │
         ▼
① 解析妙记链接，提取 session ID
   ├→ 通过 lark-minutes 获取转写内容
   └→ 获取会议元信息（主题、时间、参会人）
         │
         ▼
② AI 分析转写内容：
   ├→ 讨论要点提取
   ├→ 关键决策提取
   ├→ 待办事项提取（负责人、截止时间、内容）
   └→ 确认参会人列表
         │
         ▼
③ 展示提取结果给用户确认
         │
         ▼
④ 用户确认后：
   ├→ 生成结构化会议纪要文档
   ├→ 通过 lark-doc 创建飞书文档
   ├→ 通过 lark-wiki 归档到知识库 01-会议纪要/
   ├→ 通过 lark-base 写入待办事项表
   └→ 通过 lark-base 写入会议索引表
         │
         ▼
⑤ 通过 lark-im 推送纪要摘要到项目群
```

### 输入源B：外部转写文件

```
用户输入: claude pmo-meeting-process --file /path/to/转写.txt
         │
         ▼
① 读取本地转写文件内容
   ├→ 支持格式：.txt / .docx / .md
   ├→ 文件编码：UTF-8 / GBK 自动检测
   └→ 文件大小限制：≤ 10MB
         │
         ▼
② AI 分析转写内容：
   ├→ 会议主题识别（从文件内容提取）
   ├→ 会议日期识别
   ├→ 参会人推断
   ├→ 讨论要点提取
   ├→ 关键决策提取
   └→ 待办事项提取
         │
         ▼
③ 展示提取结果给用户确认（同上）
         │
         ▼
④ 用户确认后执行写入（同上）
         │
         ▼
⑤ 推送纪要摘要到项目群（同上）
```

## AI 提取内容规格

### 会议信息提取

| 字段 | 提取方式 | 兜底策略 |
|------|---------|---------|
| 会议主题 | AI 从转写内容识别 | 若无法识别，提示用户用 `--topic` 指定 |
| 会议日期 | AI 从上下文识别 | 若无法识别，使用当前日期 |
| 参会人 | 妙记元数据 / AI 从对话推断 | 以用户确认为准 |
| 讨论要点 | AI 分段总结，每段 1-2 句话 | — |
| 关键决策 | AI 识别决策点 | — |

### 待办提取规则

```
对转写内容做以下语义分析：

[待办内容] 从以下模式中识别：
   - "XX 需要/要负责/会处理 YY"
   - "XX 记得把 YY 完成"
   - "我来做 YY" → 说话者是负责人
   - "YY 这件事要在 DDL 前完成"
   - 明确的 "TODO: 内容"

[负责人] 从以下模式中识别：
   - @提及的人名
   - "XX做/负责/处理" 中的 XX
   - "交给XX" 中的 XX

[截止时间] 从以下模式中识别：
   - "这周五前" → 计算具体日期
   - "下周一" → 计算具体日期
   - "6月15日" → 使用具体日期
   - "尽快" → 标记为当天+3天
   - 未提及 → 标记为"待确定"，提示用户补充
```

## 写入操作细节

### 待办去重

每次写入前检查 Base 中「待办事项」表已有记录：
- 比较：待办内容（语义相似度）+ 会议来源（如相同会议）
- 相似度 >80% → 标记为重复，跳过写入
- 重复项在确认环节标注为 ⚠️ 已在待办列表中

### 知识库命名规则

```
会议纪要文档命名：
{YYYY-MM-DD}-{会议主题}.md
示例: 2026-06-09-Sprint评审.md
```

### 表写入字段映射

| 数据 | 写入表 | 对应字段 |
|------|--------|---------|
| 待办 | 待办事项 | 待办内容, 负责人, 截止日期, 状态=待处理, 优先级=P2-一般, 来源, 所属会议 |
| 会议记录 | 会议索引 | 会议主题, 会议日期, 开始/结束时间, 参会人, 记录方式, 来源文件, 纪要文档链接, 讨论要点摘要, 关键决策, 产出待办 |

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 妙记不存在或无权限 | 提示并建议使用 `--file` 方式 |
| 转写文件格式不支持 | 提示支持的格式（txt/docx/md）|
| 文件过大（>10MB） | 提示拆分或提取关键段落 |
| AI 未能提取到待办 | 提示"未识别到待办，是否仍写入会议纪要?"|
| Base 写入失败 | 纪要文档继续归档，待办暂存提示用户手动补录 |
| 知识库归档失败 | 提示用户手动上传，提供文档链接 |
