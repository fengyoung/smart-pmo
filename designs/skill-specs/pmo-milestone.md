# pmo-milestone — 里程碑管理 Skill 设计

> P1 优先级 | 里程碑查看、新增、检查、完成

---

## 调用方式

```bash
# 查看所有里程碑
claude pmo-milestone

# 检查到期里程碑（7天内到期 + 已过期）
claude pmo-milestone --check

# 新增里程碑
claude pmo-milestone --add "里程碑名称" --due 2026-07-01 --owner @张三

# 标记完成
claude pmo-milestone --complete MILE-001
```

---

## 执行流程

### 查看所有里程碑

```
① 从 Base「里程碑」表读取所有记录
② 展示结果：

   🏁 里程碑 — 智能客服平台
   ════════════════════════════
   已完成:
   ┌──────┬─────────────────┬──────────┬────────┐
   │ ID   │ 名称             │ 计划日期 │ 实际   │
   ├──────┼─────────────────┼──────────┼────────┤
   │ 001  │ 需求评审完成     │ 06-01    │ 06-01  │
   └──────┴─────────────────┴──────────┴────────┘

   进行中:
   ┌──────┬─────────────────┬──────────┬────────┬────────┐
   │ ID   │ 名称             │ 计划日期 │ 进度   │ 负责人 │
   ├──────┼─────────────────┼──────────┼────────┼────────┤
   │ 002  │ Beta版发布      │ 06-30    │ 60%    │ 李四   │
   └──────┴─────────────────┴──────────┴────────┴────────┘

   ⚠️ 已延期:
   ┌──────┬─────────────────┬──────────┬────────┐
   │ ID   │ 名称             │ 计划日期 │ 过期  │
   ├──────┼─────────────────┼──────────┼────────┤
   │ 003  │ 性能测试完成     │ 06-05    │ 4天   │
   └──────┴─────────────────┴──────────┴────────┘
```

### 到期检查 (`--check`)

```
① 从 Base 读取所有状态≠已完成的里程碑
② 筛选 ↓
   ├→ 计划日期在 7 天内 → "即将到期"
   └→ 计划日期 < 今天 → "已过期"
③ 推送提醒到项目群（通过 lark-im）：

   🏁 里程碑检查 · 2026-06-09
   ════════════════════════════
   ⚠️ 即将到期（7天内）:
   • Beta版发布 — 06-12（剩3天）@李四

   ❌ 已过期:
   • 性能测试完成 — 06-05（过4天）@王五
```

### 新增里程碑

```
① 接受参数 --add, --due, --owner
② 通过 lark-base insert_record 写入里程碑表
③ 展示确认：
   ✅ 已新增里程碑: Beta版发布 (MILE-002)
      计划日期: 2026-06-30
      负责人: 李四
```

### 标记完成

```
① 查找指定里程碑
② 通过 lark-base update_record 更新
   ├→ 状态 = "已完成"
   └→ 实际日期 = 当天
③ 展示确认：
   ✅ MILE-001 需求评审完成 已标记为完成
```

---

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 无里程碑数据 | 展示"暂无里程碑，试试 --add 新增" |
| 即将到期或过期 | 终端输出提醒（仅 --check 手动触发）|
| 里程碑 ID 不存在 | 提示"未找到该里程碑" |
