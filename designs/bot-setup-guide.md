# 飞书智能体 Bot 配置指南

> ⚠️ **已废弃（2026-06）** — 飞书计费策略调整后，Bot 模式不再使用。
> 本文档仅保留作为历史参考。当前架构为纯 Claude Code + 飞书 CLI，无 Bot 依赖。

---

## 配置流程总览

```
飞书开放平台 → 创建自建应用 → 开通权限 → 发布 → 添加智能体能力
    → 绑定 Base 数据源 → 配置定时任务 → 拉入项目群 → pmo-init 录入
```

---

## 详细步骤

### 第1步：创建自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)
2. 点击「创建应用」→「企业自建应用」
3. 填写应用名称（建议：`<项目名>-PMO-Bot`）和描述
4. 点击「确定创建」

### 第2步：开通权限

在应用详情页 →「权限管理」中开通以下权限：

| 权限 | 英文标识 | 用途 |
|------|---------|------|
| 查看、评论和编辑多维表格 | `bitable:record:read` + `bitable:record:write` | 查询/写入待办、里程碑、会议索引 |
| 获取与发送群聊消息 | `im:message` | 推送定时概览到群聊、回复 @Bot 查询 |
| 获取用户邮箱或手机号 | `contact:user.employee_id:readonly` | 识别人和待办负责人 |

### 第3步：发布应用

1. 在左侧菜单 →「版本管理与发布」
2. 创建版本 → 填写版本号（如 `1.0.0`）和更新说明
3. 点击「保存」→「申请发布」
4. 使用管理员账号审核通过

### 第4步：添加智能体能力

1. 在应用详情页 →「智能体」→「添加能力」
2. 开启「智能助手」能力
3. 配置智能体的**身份设定**（可选，建议）：

```
你是一个专业的项目管理助手 Smart-PMO。
你负责帮助项目团队管理待办事项、里程碑和会议记录。
所有数据来自项目绑定的飞书多维表格 PMO 管理台。
请用简洁清晰的语言回答，使用中文。
查询到数据时以列表或卡片形式呈现。
```

### 第5步：绑定 Base 数据源

1. 在智能体配置中 →「数据源」→「添加数据源」
2. 选择「飞书多维表格」
3. 找到 `pmo-init` 创建的 Base（命名为 `<项目名>-PMO-管理台`）
4. 勾选以下 3 张表：
   - ✅ 待办事项
   - ✅ 里程碑
   - ✅ 会议记录索引
5. 确认智能体有**读写权限**

### 第6步：配置定时任务

在智能体后台的「定时任务」中配置以下两项：

**任务1：每日待办概览推送**

| 配置项 | 值 |
|--------|------|
| 任务名称 | 每日待办概览 |
| 触发时间 | 工作日 10:00 |
| 执行动作 | 查询 Base「待办事项」表，汇总待处理、进行中、已过期的数量 |
| 推送目标 | 项目群 |
| 推送格式 | 概览卡片（包含：待处理N项、进行中N项、已过期N项）|

```
示例推送内容：
📊 项目待办概览 · 2026-06-09
待处理：5 项
进行中：3 项
⚠️ 已过期：2 项（建议尽快处理）
```

**任务2：里程碑检查推送**

| 配置项 | 值 |
|--------|------|
| 任务名称 | 里程碑到期检查 |
| 触发时间 | 工作日 09:30 |
| 执行动作 | 查询 Base「里程碑」表，检查7天内到期或已过期的里程碑 |
| 推送目标 | 项目群 |
| 推送格式 | 检查卡片（包含：即将到期项、已过期项）|

```
示例推送内容：
🏁 里程碑检查 · 2026-06-09
⚠️ 即将到期（7天内）：
  • Beta版发布 — 2026-06-12（剩3天）
❌ 已过期：
  • 需求评审 — 2026-06-05（已过4天）
```

### 第7步：将 Bot 拉入项目群

1. 在应用详情页 →「设置」→「机器人」
2. 确保「机器人」能力已启用
3. 打开项目群聊 → 群设置 → 添加机器人 → 选择刚创建的应用

### 第8步：在 pmo-init 中录入

运行 `pmo-init` 时按提示录入：

```bash
# 进入项目目录
cd ~/projects/smart-customer-platform
claude

# 启动初始化（或在初始化过程中录入）
# 会提示输入以下 Bot 信息：
#   appId: cli_abc123
#   appSecret: xxxxxxxxxx
#   项目群 ID: oc_xxxxx
```

## 验证配置

配置完成后，在群聊中测试：

| 测试项 | 测试方式 | 预期结果 |
|--------|---------|---------|
| @Bot 查询 | `@SmartPMO 我的待办有哪些` | Bot 回复待办列表 |
| @Bot 新增 | `@SmartPMO 创建待办：完成方案评审，负责人@小王` | 待办写入 Base |
| 定时推送 | 等待 10:00 或手动触发 | 群内收到概览卡片 |

---

## 注意事项

1. **权限审批**：首次发布需要管理员审批，建议提前准备好
2. **Base 操作权限**：智能体的 Base 读写权限必须和 `pmo-init` 创建的是同一个 Base
3. **多项目隔离**：每个项目独立 Bot，不同项目群的 Bot 不能互查对方数据
4. **定时任务测试**：配置定时任务后，建议先手动测试一次查询逻辑
5. **更新配置**：如果 Base 表结构有调整，需要在智能体数据源中重新同步
