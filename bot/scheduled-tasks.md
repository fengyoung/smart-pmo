# 飞书智能体 Bot 定时任务配置

> 配置智能体后台的定时推送任务

---

## 任务1：每日待办概览

| 配置项 | 值 |
|--------|------|
| **任务名称** | 每日待办概览 |
| **触发时间** | 工作日 10:00 |
| **执行逻辑** | 查询多维表格「待办事项」表，分类统计 |
| **推送目标** | 项目群 |

### 查询内容

```sql
-- 概念性查询（飞书智能体配置时使用自然语言）
统计当前「待办事项」表中：
1. 状态="待处理" 的记录数
2. 状态="进行中" 的记录数
3. 截止日期<今天 且 状态≠"已完成"且≠"已取消" 的记录数
```

### 推送格式

```
📊 [项目名] 待办概览 · YYYY-MM-DD
═══════════════════════════════
📋 待处理：N 项
🔄 进行中：N 项
⚠️ 已过期：N 项
```

## 任务2：里程碑到期检查

| 配置项 | 值 |
|--------|------|
| **任务名称** | 里程碑到期检查 |
| **触发时间** | 工作日 09:30 |
| **执行逻辑** | 查询多维表格「里程碑」表，筛选即将到期或已过期 |
| **推送目标** | 项目群 |

### 查询内容

```sql
-- 概念性查询
从「里程碑」表中查询：
1. 计划日期在 7 天内 且 状态≠"已完成" → 即将到期
2. 计划日期<今天 且 状态≠"已完成" → 已过期
```

### 推送格式

```
🏁 里程碑检查 · YYYY-MM-DD
═══════════════════════════════
⚠️ 即将到期（7天内）：
• [名称] — YYYY-MM-DD（剩N天）@[负责人]

❌ 已过期：
• [名称] — YYYY-MM-DD（过N天）@[负责人]
```

---

## 配置步骤

在飞书开放平台智能体后台：

1. 进入「智能体」→「定时任务」
2. 添加新任务，配置触发时间和执行动作
3. 执行动作为：查询 Base + 组装消息 + 发送到群聊
4. 保存并启用

> 不同飞书版本可能操作路径不同，以实际后台为准。
