## 1. 概述

Smart-PMO 是一套项目管理 Skill 集合，在 Claude Code 中通过自然语言指令驱动，自动操作飞书的知识库、多维表格、群聊等资源，覆盖项目管理全流程。

### 1.1 核心能力

| 功能 | Skill | 说明 |
|------|-------|------|
| 项目初始化 | `pmo-init` | 自动创建知识空间、多维表格 Base、注册配置 |
| 会议处理 | `pmo-meeting-process` | 从妙记或转写文件提取纪要/待办/决策；支持 --sub-project 跨项目周会提取 |
| 群聊提取待办 | `pmo-todo-from-chat` | 从所有项目群消息中识别待办，确认后写入 Base；多群独立推进读取位置 |
| 待办跟进 | `pmo-todo-followup` | 查看、标记完成、修改待办；行序号当次有效；跨项目操作支持 --project |
| 文档归档 | `pmo-archive` | 上传文件到知识库 |
| 里程碑管理 | `pmo-milestone` | 查看、新增、检查到期里程碑；行序号当次有效 |
| 周报生成 | `pmo-weekly-report` | 汇总本周数据生成周报，重复生成自动追加序号；支持 --send 推送到群 |
| 多项目管理 | `pmo-list/use/pin/dashboard` | 切换/关注/概览多个项目；dashboard 为项目风险视角 |
| 跨表搜索 | `pmo-search` | 搜索待办/里程碑/会议记录，结果附 Base 记录直达链接 |
| 数据导出 | `pmo-export` | 导出 Base 三张表为 CSV / JSON / XLSX |
| 项目信息 | `pmo-info` | 查看项目详情、成员、统计、资源链接 |
| 今日概览 | `pmo-today` | 个人视角：展示我负责的今日截止/过期待办；--all-members 切换全员视角 |
| 风险扫描 | `pmo-risk-scan` | 自动检测过期待办率、会议断层、里程碑延期等风险信号，输出四级评级 |
| 主动提醒 | `pmo-notify` | 向项目群推送过期待办和即将到期里程碑提醒；支持 --dry-run 预览 |
| 统计分析 | `pmo-stats` | 按周/月统计待办趋势、负责人贡献、里程碑完成率；ASCII 趋势图 |
| 批量导入 | `pmo-import` | 从 CSV / XLSX / Markdown 表格批量导入待办和里程碑到 Base |
| 会前准备 | `pmo-meeting-prep` | 拉取上次遗留+过期待办+临近里程碑，自动生成会前议程文档 |
| 变更日志 | `pmo-changelog` | 汇总已完成待办和里程碑，AI 生成 Changelog，可归档或导出 |
| 燃尽图 | `pmo-burn-down` | 基于里程碑关联待办生成 ASCII 燃尽图，推算预计完成日期 |
| 项目复盘 | `pmo-retro` | 基于会议纪要和待办数据 AI 辅助生成复盘报告，行动计划可写入 Base |
| 健康检查 | `pmo-health` | 诊断配置完整性、Base/Wiki 连通性、队列积压，输出健康报告 |
| 站会速报 | `pmo-standup` | 生成个人视角站会速报（昨日/今日/阻塞），可直接推送到群 |
| 轻量周报 | `pmo-weekly-digest` | 生成消息卡片推送到群，不创建文档，与 pmo-weekly-report 互补 |

### 1.2 架构

```
Claude Code (本地)
    │
    │  AI 密集型操作
    │  会议/待办/归档/周报
    │
    └──────────┬──────────┘
               │
     ┌─────────▼──────────┐
     │  统一数据源          │
     │  Base + 知识库 + 群  │
     └────────────────────┘
```

## 2. 快速开始

### 2.1 前提条件

- Claude Code 已安装并可用
- 飞书 CLI（`@larksuite/cli`）已安装并完成认证

### 2.2 初始化第一个项目

```bash
# 启动 Claude Code
claude

# 执行项目初始化（交互式）
claude pmo-init
```

按提示依次输入：项目名称、项目经理、核心成员、项目群。

### 2.3 验证安装

```bash
# 查看当前项目
claude pmo-list
```

如果显示已注册的项目信息，说明安装成功。

## 3. 项目管理

### 3.1 查看所有项目

```bash
claude pmo-list

# JSON 格式输出（脚本/管道消费）
claude pmo-list --json
```

输出示例：
```
📋 所有已注册项目：

  → XRay拆修检测2026 (XRay) | active | PM: 邹晨风 ※ 关注

──────────────────────────────
使用 pmo-use <项目名> 切换项目
使用 pmo-pin <项目名>  关注项目
```

### 3.2 切换当前项目

```bash
# 切换到指定项目（当前会话有效）
claude pmo-use XRay

# 全局持久化切换（其他终端也生效）
claude pmo-use XRay -g
```

切换后，所有后续 Skill 操作的 Base、知识库、群都跟着切换。

### 3.3 关注/取消关注项目

```bash
# 关注一个或多个项目（用于 dashboard 聚合展示）
claude pmo-pin XRay

# 取消关注
claude pmo-unpin XRay
```

### 3.4 多项目概览（项目风险视角）

```bash
# 查看所有关注项目的待办和里程碑概览
claude pmo-dashboard
```

展示全员维度的各项目待处理/过期待办和即将到期里程碑，有告警项时自动提示下钻。

> **与 `pmo-today` 的区别：** `pmo-dashboard` 是项目风险视角（全员），`pmo-today` 是个人视角（我今天要做什么）。

> 值日场景可开启定时巡检：`/loop 30m pmo-dashboard`

## 4. 会议处理

### 4.1 通过飞书妙记处理

```bash
claude pmo-meeting-process --minutes https://meetings.feishu.cn/xxx
```

Claude Code 会自动：
1. 通过 `lark-minutes` 获取转写内容
2. AI 分析提取讨论要点、关键决策、待办事项
3. 展示结果供您确认
4. 确认后：
   - 生成结构化会议纪要 → 归档到知识库 `01-会议纪要/`
   - 待办写入 Base → 关联会议记录
   - 会议记录写入 Base 索引表

### 4.2 通过外部转写文件处理

```bash
# 录音笔导出的转写文本
claude pmo-meeting-process --file /path/to/转写文本.txt
```

支持格式：`.txt` / `.docx` / `.md`，大小限制 ≤ 10MB。

### 4.3 选择性写入模式

确认界面提供 6 种操作选项：

| 选项 | 行为 |
|------|------|
| 全部写入 | 完整流程：纪要→知识库→索引→待办→关联 |
| 仅写待办 | 跳过纪要生成，只写 Base 待办+会议索引 |
| 仅写纪要 | 只生成文档+归档知识库，不写 Base |
| 选择写入 | 指定待办编号（如 `1,3,5-7`），只写入选中项 |
| 修改 | 调整内容后重新确认 |
| 取消 | 放弃操作，自动缓存草稿 |

### 4.4 独立创建会议索引

```bash
# 为已有待办建立会议索引和双向关联
claude pmo-meeting-process --index-only "Sprint评审" "2026-06-09"
```

### 4.5 多项目周会提取

```bash
# 从跨项目周会中提取 XRay 子专题
claude pmo-meeting-process --file /path/to/周会.txt --sub-project XRay
```

### 4.6 手动指定会议参数

```bash
claude pmo-meeting-process --minutes <链接> --topic "Sprint评审" --date 2026-06-09
```

### 4.7 智能增强（v1.5.0 新增）

pmo-meeting-process 新增 4 项智能增强能力，可提升会议纪要提取的准确性和便捷性：

**① 文件名自动解析**
外部转写文件命名符合 `YYYYMMDD-{主题}` 格式时，自动提取日期和主题，无需手动指定参数：
```bash
# 文件名 20260410-XRay安卓150款机型推进计划.txt
# → 自动识别日期=2026-04-10，主题="XRay安卓150款机型推进计划"
claude pmo-meeting-process --file 20260410-XRay安卓150款机型推进计划.txt
```

**② ASR 校正表自动加载**
在项目知识库 `05-项目资料/` 下放置 `ASR校正表.md`，pmo-meeting-process 会自动读取用于校正语音转写中的误识别（人名、术语等）。可通过 `claude pmo-meeting-process --file <文件>` 直接体验。

**③ 历史上下文注入**
处理会议时自动拉取最近 2 条历史会议记录的讨论要点，作为上下文注入 AI 提取环节，提升待办识别的连贯性。

**④ 结构化 Prompt 模板**
采用三段式 Prompt（背景上下文 → 任务说明 → 输出格式），确保每次提取结果格式一致、内容完整。

> 💡 ASR 校正表可通过修改知识库 `05-项目资料/ASR校正表.md` 文档实时更新，下次处理会议时自动生效。

## 5. 待办管理

### 5.1 查看待办

```bash
# 全部待办
claude pmo-todo-followup

# 只看我的
claude pmo-todo-followup --mine

# 只看已过期
claude pmo-todo-followup --overdue

# 按状态筛选
claude pmo-todo-followup --status 待处理
```

### 5.2 标记完成 / 修改

```bash
# 标记完成（TODO-ID 或列表行序号均可）
claude pmo-todo-followup --complete TODO-003
claude pmo-todo-followup --complete 3

# 转交待办
claude pmo-todo-followup --modify TODO-003 --assign @李四

# 修改截止日期
claude pmo-todo-followup --modify TODO-003 --due 2026-06-20
```

> **行序号说明：** 行序号在当次展示的列表中有效。切换项目或任何写操作后序号失效；未先查看列表时直接使用序号，会自动重新拉取列表并提示映射已刷新。

### 5.2.1 跨项目直接操作（--all 模式）

```bash
# 先查看所有项目待办
claude pmo-todo-followup --all

# 指定项目名，无需切换上下文
claude pmo-todo-followup --all --complete TODO-003 --project XRay
claude pmo-todo-followup --all --modify TODO-003 --due 2026-06-20 --project XRay
```

### 5.3 从群聊提取待办

```bash
claude pmo-todo-from-chat
```

执行后：
1. 并发读取项目配置中 **所有群**（`chatIds`）自上次处理以来的新消息
2. AI 识别潜在待办（3 层去重：消息 ID 去重 + 内容语义去重 + 用户确认）
3. 展示结果供您选择写入，每条标注来源群名
4. 写入后各群的读取位置**独立**推进：全部成功才更新该群位置；部分失败时保留消息窗口，下次自动重试

## 7. 里程碑管理

```bash
# 查看所有里程碑（含行序号，已完成分组默认折叠）
claude pmo-milestone

# 查看全部（含所有已完成）
claude pmo-milestone --all

# 检查到期情况（7天内到期 + 已过期，含逾期天数）
claude pmo-milestone --check

# 新增里程碑（完整参数）
claude pmo-milestone --add "Beta版发布" --due 2026-06-30 --owner @张三
claude pmo-milestone --add "需求评审" --due 2026-06-20 --owner @李四 --progress 30 --status 进行中 --desc "验收标准：..."

# 标记完成（MILE-ID 或查看列表后的行序号均可，自动填实际日期+补齐进度）
claude pmo-milestone --complete MILE-001
claude pmo-milestone --complete 2

# 修改里程碑（支持行序号，支持多字段同时修改）
claude pmo-milestone --modify 1 --progress 75
claude pmo-milestone --modify MILE-001 --due 2026-07-15
claude pmo-milestone --modify MILE-001 --status 已延期
claude pmo-milestone --modify 3 --owner @王五 --desc "更新后的描述"
```

> **行序号说明：** 行序号在当次展示列表中有效，切换项目或任何写操作后失效；未先查看列表时直接使用序号，会自动重新拉取并提示映射已刷新。

## 8. 周报生成

```bash
# 生成本周周报
claude pmo-weekly-report

# 预览草稿，不写入（dry-run）
claude pmo-weekly-report --dry-run

# 生成指定周
claude pmo-weekly-report --week 2026-06-01

# 不对比上周数据
claude pmo-weekly-report --no-compare

# 生成并推送摘要卡片到项目群
claude pmo-weekly-report --send

# 推送到指定群
claude pmo-weekly-report --send --chat-id <chat_id>
```

流程：
1. 去重检测（已生成过本周周报时提示是否继续；继续则新文档自动追加 `-2`、`-3` 序号，不覆盖旧文档）
2. 从 Base 并行统计本周会议、待办完成率、里程碑状态（与上周对比，估算数据标注 `*估算`）
3. AI 整理周报内容（含风险识别）
4. 展示草稿供确认（支持修改反馈循环）
5. 确认后 → 生成文档 → 归档到 `02-周报/`
6. 若指定 `--send`：推送摘要卡片 + 文档链接到目标群
7. `--dry-run` 模式仅预览，不创建文档、不归档、不推送

## 9. 搜索

```bash
# 在当前项目搜索
claude pmo-search <关键词>

# 限定搜索范围
claude pmo-search <关键词> --in todos        # 仅待办
claude pmo-search <关键词> --in milestones    # 仅里程碑
claude pmo-search <关键词> --in meetings      # 仅会议

# 跨所有项目搜索
claude pmo-search <关键词> --all

# 限定时间和负责人
claude pmo-search <关键词> --since 2026-05-01 --until 2026-06-01
claude pmo-search <关键词> --owner @张三

# 调整每表返回条数（默认 20，最大 100）
claude pmo-search <关键词> --limit 50
```

结果达到上限时会提示总匹配数，建议缩小关键词或时间范围。每条结果附 **Base 记录直达链接**，点击可直接跳转到对应记录；会议结果同时附纪要文档链接。

## 10. 数据导出

```bash
# 导出全部三张表（默认 CSV）
claude pmo-export

# 指定格式
claude pmo-export --format csv
claude pmo-export --format json

# 仅导出指定表
claude pmo-export --table todos
claude pmo-export --table milestones
claude pmo-export --table meetings

# 指定输出目录
claude pmo-export --output ~/Desktop/pmo-backup
```

导出文件说明：
- CSV 采用 **UTF-8 BOM** 编码，可直接用 Excel 打开不乱码
- 人员/关联字段自动展开为可读值（姓名、自动编号 ID）
- 同时输出 `export_meta.json` 汇总文件（导出时间、各表记录数）

## 12. 今日概览

```bash
# 当前项目今日概览（个人视角：仅展示我负责的待办）
claude pmo-today

# 切换到全员视角（展示所有成员的今日待办）
claude pmo-today --all-members

# 跨所有关注项目（仍为个人视角）
claude pmo-today --all

# 跨关注项目 + 全员视角
claude pmo-today --all --all-members
```

**个人视角（默认）：** 仅展示当前登录用户负责的今日截止/过期待办；里程碑和会议始终展示全员数据。

**全员视角（--all-members）：** 展示所有成员的今日截止/过期待办，含负责人信息。

展示内容：
- 今日截止待办（按优先级排序）
- 已过期待办
- 7 天内到期里程碑
- 今日会议（含纪要链接）

今天什么都没有时显示待处理总数，不打扰。

> **与 `pmo-dashboard` 的区别：** `pmo-today` 是个人视角（我今天要做什么），`pmo-dashboard` 是项目风险视角（全员告警统计）。

## 13. 风险扫描

```bash
# 扫描所有关注项目的风险
claude pmo-risk-scan

# 仅扫描指定项目
claude pmo-risk-scan --project XRay

# 仅显示高风险项目（🟡 及以上）

# 生成风险报告推送到项目群

```

**风险指标：** 过期待办率、严重过期数、会议断层（14天未开会）、里程碑延期率、僵尸待办（30天无进展）。

**风险等级：** 🟢健康 → 🟡关注 → 🟠警告 → 🔴危险

## 24. 常见问题

### 24.1 如何添加新项目？

```bash
claude pmo-init
```
按提示交互式创建。

### 24.2 如何添加或修改项目成员？

```bash
claude pmo-use --add-member --name 张三 --role 开发
claude pmo-use --remove-member --name 张三
claude pmo-use --list-members
```

### 24.3 配置存储在哪里？

所有配置在 `~/.smart-pmo/` 目录下（家目录，不在 Git 管理范围内）：
- `registry/<项目名>.json` — 项目配置
- `current` — 当前项目指针
- `pinned` — 关注项目列表

### 24.4 如何切换到另一个项目？

```bash
claude pmo-use <项目名>
```

### 24.5 妙记方式和文件方式有什么区别？

| 方式 | 命令 | 适用场景 |
|------|------|---------|
| 飞书妙记 | `--minutes <链接>` | 飞书会议自动录制 |
| 外部文件 | `--file <路径>` | 录音笔转写、线下会议笔记 |

### 24.6 支持哪些文件格式？

会议处理：`.txt` / `.docx` / `.md`（≤10MB）
文档归档：`.docx` / `.pdf` / `.xlsx` / `.pptx` / `.png` / `.jpg` / `.md` / `.txt`（≤50MB）

### 24.7 遇到 pmo-* 命令异常怎么办？

首先运行健康检查定位问题：

```bash
claude pmo-health
```

常见问题：
- Base 连接失败 → 检查 `baseAppToken` 是否有效
- 配置不完整 → 重新运行 `pmo-init` 修复
- 待处理队列积压 → 运行 `pmo-todo-followup` 自动重试
